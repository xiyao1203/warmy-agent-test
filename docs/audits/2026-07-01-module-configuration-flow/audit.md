# Agent 测试全链路配置审查

- 日期：2026-07-01
- 范围：智能体、数据集/用例、测试计划、执行、环境凭证、评分器、实验、人工审核、安全测试、发布门禁
- 目标：确认用户能否理解必填内容，并验证每项配置是否进入真实执行链
- 截图：本目录 `01`—`12`，均来自本次本地产品走查

## 流程步骤与健康度

1. 智能体列表（`01-agents.png`）：较差。只能看到名称、类型和时间，缺少接入健康、当前发布版本与最近验证状态；窄视口下创建按钮文字换行。
2. 智能体版本（`02-agent-version-dialog.png`）：不可用。页面只暴露 API 地址、模型、超时；后端已有的协议、请求映射、响应映射、代码/Prompt/知识版本、工具、步数、成本等没有配置入口，也没有“测试连接”。
3. 数据集入口（`03-datasets.png`）：不可用。全局“导入”没有目标数据集/草稿版本上下文。
4. 用例导入（`04-import-dialog.png`）：不可用。点击后弹窗立即关闭；没有模板下载、字段说明、示例、预检、逐行错误或导入统计。
5. 环境列表（`05-environments.png`）：较差。只展示名称/类型，用户看不到协议、变量、凭证引用或可用状态。
6. 环境创建（`06-environment-dialog.png`）：不可用。只填写名称和描述，实际 `config` 始终为空；页面宣称支持凭证/Mock/沙箱，但没有配置控件。
7. 测试计划（`07-test-plans.png`）：较差。资产关联隐藏在详情页，缺少就绪状态和“哪些前置项尚未配置”的解释。
8. 评分器（`08-scorers.png`）：不可用。只能填类型、权重、阈值，无法配置规则表达式、模型裁判、参考答案字段或输出提取；运行 Worker 不消费评分器。
9. 实验对比（`09-experiments.png`）：较差。创建实验要求手填 Run UUID，没有候选运行筛选、可比性检查和基线/候选语义。
10. 人工审核（`10-reviews.png`）：断链。审核台本身可操作，但自动收集器没有接入 Run/评分/安全结果，任务不会自然产生。
11. 安全测试（`11-security.png`）：断链。要求再次手填 Agent URL，没有引用 AgentVersion、环境凭证、数据集/策略或来源 Run；历史任务可长期停在 pending。
12. 发布门禁（`12-gates.png`）：虚假评估。页面不从 Run/实验/安全/人工审核聚合指标，当前前端以固定通过率 `0.85` 和 `critical_passed=true` 调用评估。

## 代码侧根因

1. `SqlAlchemyRunSource` 把测试计划配置写入 `config_snapshot`，但 API Runner 把它当作 Agent 配置读取 `url/mode/headers`；真实 Run 的核心快照结构不一致。
2. Agent 版本后端字段与前端表单不一致；运行时又期待另一套 `url` 字段，缺少统一、版本化的 Agent 调用协议。
3. 环境模板 `config` 是无 Schema 的 JSON，前端创建时不提交配置，RunSource 也完全不读取 `environment_template_id`。
4. 导入解析器对非对象 `input` 静默转成 `{}`，对部分错误类型静默转成空值；没有可共享的导入规范或预检契约。
5. 测试计划的 `scorers`、`retry_policy`、`cost_budget`、`release_gate` 多为无类型 JSON；页面缺少真实资产选择，其中 `release_gate` 不是门禁 ID。
6. RunCase 快照只带 input/assertions，丢失 scorer、expected_outcome、security_policy、initial_state 等用例语义。
7. API Runner 只执行 `contains/exact` 两种断言，不执行评分器、预算、安全策略或审核收集。
8. 实验直接比较两个 Run，但没有保证 Agent、数据集或用例集合可比。
9. 安全扫描、人工审核和发布门禁是各自孤立的 CRUD/动作页面，没有统一的 Evaluation/ReleaseDecision 聚合链。

## 可见的无障碍与响应式风险

- 约 493px 可用宽度下，多数页面仍保留桌面侧栏，标题和主操作按钮重叠或被裁剪。
- 图标操作缺少持续可见的文字说明，配置字段缺少帮助文本和示例。
- 空状态只说明“暂无”，没有显示前置条件和下一步。
- 截图只能证明可见问题；键盘顺序、焦点回收、读屏状态播报仍需实现后做自动和人工验证。

## 审查结论

当前产品不是“文档缺少解释”，而是资产模型、配置 Schema、运行快照和下游消费没有形成同一条契约。必须先统一真实测试执行模型，再重做各模块的表单和关联；仅补 tooltip 或帮助文案无法修复。
