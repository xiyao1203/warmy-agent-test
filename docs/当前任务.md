## 目标

交付 M3 的首个完整纵向闭环：用户可从运行中心基于已发布测试计划创建运行，查看 Run/RunCase 进度、结果与规范化 Trace，并取消运行；控制面通过编排端口启动 Temporal Workflow，API Runner Worker 不直接访问业务数据库。

## 本次范围

- `apps/control-api/`：Run/RunCase 领域、迁移、应用服务、项目隔离 API、SSE 基础能力。
- `workers/api-runner/`：Generic HTTP Agent Adapter、Activity 和 Temporal Workflow。
- `apps/web/`：运行中心、运行详情、进度、取消、结果和 Trace 基础展示。
- `packages/generated-api-client/`：运行 API 契约生成。
- `infra/compose/`：仅在 Temporal 本地配置确有缺口时修改。
- `docs/api/`、`docs/security/dependencies/`、`docs/superpowers/plans/`：契约、依赖核对和实施记录。
- `docs/当前任务.md`、`docs/开发进度与变更记录.md`。

## 关键约束

- 所有运行数据必须关联并校验 `project_id`。
- Run 只能引用已发布的 AgentVersion、DatasetVersion 和 TestPlanVersion。
- 同一项目、同一 `Idempotency-Key` 只能创建一个 Run。
- `FAILED`、`ERROR`、`CANCELLED` 必须语义分离。
- Workflow 保持确定性，外部 I/O 只在 Activity 中执行。
- API Runner Worker 不得连接或导入业务数据库/控制面内部模块。
- Agent 流式对话三栏工作区继续保留，不被运行中心导航覆盖。

## 验收方式

- 后端：领域单元测试、API 契约测试、项目隔离测试、架构测试、Ruff、mypy。
- 数据库：空库迁移、升级、约束和索引检查（环境不可用时记录未验证与风险）。
- Worker：Fake Target、Workflow replay、重试、超时、取消、幂等测试。
- 前端：运行中心组件测试、关键 E2E、格式化、ESLint、TypeScript、Production Build。
- 契约：OpenAPI 与 TypeScript Client 可重复生成且无漂移。
