# Agent 测试完整链路操作手册

## 1. 准备可执行资产

1. 在“智能体”创建 Agent 和版本，填写协议、真实端点、请求模板、响应路径、超时与凭证引用。连接验证通过后发布版本。
2. 在“环境与凭证”录入加密凭证，再配置变量、公开 Header、初始状态、沙箱参数和凭证注入位置。API 永不返回明文，Worker 仅在执行 Activity 内解密。
3. 在“测试用例”创建数据集草稿，按[导入格式](../test-case-import-format.md)预检并导入，确认后发布版本。
4. 在“评分器”创建规则、参考答案或模型裁判评分器，先使用“试评”验证配置。模型裁判必须引用项目模型配置并真实调用模型运行时。

## 2. 组装并执行计划

1. 创建测试计划版本，选择已发布 Agent、数据集、环境以及评分器、安全策略、审核策略和发布门禁。
2. 配置并发、单用例超时、最大重试、通过阈值和费用预算。若不评分，必须显式启用“仅观察”；否则计划不能发布。
3. 发布计划并在“测试执行”创建 Run。控制面将所有引用解析成不可变快照；Worker 按协议调用被测 Agent，并回传每条用例的输出、脱敏 Trace、耗时和错误分类。

## 3. 消费真实证据

- 运行完成后形成 RunEvaluation 和逐用例 Score；失败或低置信度结果自动进入人工审核。
- “实验对比”只能选择同项目、同计划版本且已完成的真实 Run。
- “安全测试”选择已发布 Agent 版本和环境/策略，不接受浏览器提交任意目标 URL；扫描结果关联 Run 和发现项。
- “发布门禁”只接收 Run（可选实验）引用，由服务端读取评测、关键用例、安全发现和待审核项生成 ReleaseDecision，客户端不能提交通过率或安全事实。
- 测试 Agent 使用与控制台相同的应用服务和资产 ID；其创建、执行、安全扫描与门禁操作产生同一套平台记录，不维护第二份演示数据。

## 4. 故障定位

- 计划无法发布：按就绪检查补齐未发布资产、评分器或“仅观察”声明。
- Run 返回 503：Temporal/API Runner 未就绪；平台不会写入假成功 Run。
- Agent 调用失败：查看错误分类和脱敏 Trace，检查端点、请求/响应映射及凭证注入位置。
- 模型评分失败：检查项目模型配置、默认模型、Model Runner 和供应商配额；平台不会降级为 Mock 分数。
