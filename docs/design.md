# AgentTest 工作台设计系统

> 当前生效版本：2026-07-17。详细重构规格见
> `docs/superpowers/specs/2026-07-17-glm-coding-frontend-redesign-design.md`。

## 1. 设计方向

AgentTest 是面向专业测试工程师的高密度工作台。视觉语言参考 GLM Coding 的中性表面、紧凑布局、单色导航和珊瑚红强调色，但不复制第三方商标、图形资产、文案或业务页面。

设计原则：

- 内容和执行状态优先，应用框架保持克制。
- 颜色只表达品牌动作或语义状态，不作为状态的唯一载体。
- 表格、Trace、对话和结构化配置获得主要屏幕空间。
- 页面使用统一的语义 Token 和共享原语，不在业务模块写产品色值。
- 浅色、深色、跟随系统三种主题具有同等功能和可读性。

## 3. 排版与密度

- 字体：中文系统无衬线优先；数字使用 `tabular-nums`；日志、请求和代码使用等宽字体。
- 字号：12 / 13 / 14 / 16 / 20 / 24 / 32px；页面标题通常为 20–24px。
- 间距：4px 基础网格；常用页面间距为 12 / 16 / 20 / 24px。
- 控件：紧凑 32px、标准 36px、强调 40px。
- 圆角：6 / 8 / 12px；胶囊仅用于徽标、状态和身份。
- 阴影：仅用于 Dropdown、Popover、Drawer、Dialog 和 Toast。
- 动效：快速 120ms、标准 180ms，并遵守 `prefers-reduced-motion`。

## 4. 应用框架

桌面端采用分组侧栏和紧凑顶部栏：

- 工作台：测试 Agent、概览。
- 测试资产：智能体、测试用例、测试计划。
- 执行中心：测试执行、环境与凭证、浏览器实例。
- 评测与治理：评分器、实验对比、人工审核、安全测试、发布门禁。
- 系统设置：模型配置；用户与权限仅超级管理员可见。

顶部栏包含项目切换、面包屑、`Cmd/Ctrl+K` 命令面板、快速创建、运行状态、主题、帮助、通知和账户。390px 等窄屏将导航移入可访问抽屉，禁止页面级水平滚动。

## 5. 页面模式

### Agent 工作区

宽屏使用“会话历史 / 对话 / 计划与证据”结构。消息使用低装饰内容流，确认、执行、结果和失败采用明确的专用状态组件。

### 列表与详情

使用紧凑页头、筛选工具条、左对齐决策表格、批量操作和分页。表格可局部横向滚动，主要操作不得隐藏在不可发现的菜单中。

### 运行与结果

先展示状态、进度、通过/失败/错误/审核中、成本和剩余时间，再提供结果导航、输入输出、Trace、产物、评分和审核意见。`FAILED` 与 `ERROR` 必须使用不同文字标签。

## 6. 主题契约

- 用户选择为 `light`、`dark` 或 `system`，写入 `localStorage.theme`。
- `system` 必须监听 `prefers-color-scheme` 的实时变化。
- 根布局在 React 水合前解析并应用有效主题，避免首屏错色闪烁。
- 根元素的 `data-theme` 与 `color-scheme` 必须和有效主题一致。

## 7. 组件约束

- Button、Input、Select、Badge、Table、Dialog、Drawer、Dropdown、Toast 和卡片只消费语义 Token。
- 主按钮为紧凑圆角矩形，不使用大胶囊、装饰性渐变或发光效果。
- 导航图标使用单色 Lucide 图标；禁止彩色 3D 图标作为应用导航。
- 每个图标按钮具有可访问名称和可见焦点。
- Feature 通过公开出口依赖共享 UI；共享 UI 不反向依赖 Feature。

## 8. 验证要求

每次影响设计系统或应用框架的变更至少验证：

1. Prettier、ESLint、TypeScript 和 Vitest。
2. 关键 Playwright 流程、Next.js 构建、架构边界和 Bundle Budget。
3. 390 / 1280 / 1440 / 1920 宽度。
4. 浅色、深色和跟随系统；键盘焦点、对比度、抽屉和对话框。
5. 路由、权限、API 契约和专业测试用例字段没有回归。
