# Warmy Agent Test 设计系统

> 当前生效版本：2026-07-20。完整方案与验收标准见
> `docs/superpowers/specs/2026-07-20-precision-motion-platform-ui-design.md`。

## 1. Precision Motion

Warmy Agent Test 是面向专业测试团队的高密度工作台。平台采用 **Precision Motion** 视觉语言：中性、精密、紧凑，动效只用于反馈操作、表达状态和说明空间关系。

参考边界：

- Uiverse：按钮、菜单、Hover、Pressed 和 Loading 的触感。
- Aceternity UI：边界响应、抬升层次和内容进入的空间感。
- React Bits：数字更新、状态切换和图标语义动画。
- 不复制第三方页面、品牌资产或完整组件，不使用大面积渐变、霓虹、玻璃、光球、星芒、抽象 AI 图标或无意义循环动画。

## 2. 视觉基础

### 颜色与表面

- 沿用中性灰画布与珊瑚红主色；主色只用于主要操作、焦点和选中状态。
- 成功、警告、失败、信息使用语义 Token，并同时提供图标或文字。
- 页面分区优先使用留白和分隔线；卡片仅用于重复资源、指标、模态和真实工具表面。
- 阴影只用于 Hover 抬升、Dropdown、Popover、Dialog、Drawer 和 Toast。
- 浅色和深色两种主题保持等价功能与可读性；主题只由用户显式切换。

### 排版与密度

- 中文使用系统无衬线字体，数字使用 `tabular-nums`，日志和代码使用等宽字体。
- 页面标题通常为 20px-24px；落地页产品名可使用 40px-58px。
- 4px 间距网格；常用间距为 12px、16px、20px、24px。
- 控件高度为 32px、36px、40px；工作区优先使用 32px 或 36px。
- 圆角以 6px、8px 为主，普通卡片不超过 8px；胶囊仅用于状态、筛选和身份。
- 图标统一使用 `lucide-react`，光学尺寸 17px、线宽约 1.75、承载区域 24px。
- 字距固定为 0，不使用 Emoji、彩色 3D 图标或临时手绘符号。

### 动效节奏

- 微交互 120ms-160ms，菜单与轻浮层 140ms-180ms，状态切换 180ms。
- Dialog 220ms，Drawer 260ms，空间变化不超过 280ms。
- 只有运行中状态可持续循环；完成描线、数字更新和异常提醒只执行一次。
- `prefers-reduced-motion` 下关闭非必要动画，保留颜色、文字和结构状态。

## 3. 落地页与登录

- H1 固定为 `Warmy Agent Test`，价值主张放在支持文案中。
- 首屏采用单列产品叙事与全宽真实测试运行场景，不使用左右分栏、营销 Bento、虚构 Logo、价格或公开注册。
- 主视觉展示资源版本、运行状态、评分、审核和门禁证据，所有视口都露出下一段内容。
- 未登录主操作为“登录”；登录成功仅关闭 Dialog 并停留落地页，主操作更新为“进入工作台”。
- 用户再次点击“进入工作台”时才导航；已有登录态访问 `/` 或 `/login` 仍停留落地页。
- 次操作只滚动到产品证据，不打开登录弹窗。
- 产品文案只描述业务事实、下一步和必要风险，不展示设计思考、实现范围或未提供能力的内部说明。

## 4. 应用框架

- 桌面侧栏展开宽度 208px，收起宽度 56px，导航行高 34px。
- 侧栏只承载导航，不展示数量、通过率、运行状态或其他统计。
- 当前项使用 2px 状态边、轻背景和文字强调；图标放在稳定的 24px 承载区。
- 收起状态使用 Tooltip 展示名称，不依赖原生 `title`。
- 顶部栏保留项目切换、全局搜索、快速创建、运行入口、主题、帮助、通知和账户菜单。
- 移动端导航进入可访问 Drawer，桌面收起状态沿用现有偏好存储。

## 5. 组件规范

### 紧凑统计卡

统一采用“图标结合状态边”的 C 方案：

- 桌面高度 88px，移动端 84px；桌面四列，移动端两列。
- 17px Lucide 图标与 2px 状态边共同表达语义，不使用彩色方形或圆形图标底座。
- Hover 时状态边展开、图标轻移、顶部短高光出现，快捷入口在固定区域显现，不改变尺寸。
- 支持默认、Hover、Focus、选中、Loading、空数据、错误、无权限、运行中和不可点击状态。
- 不可点击卡片不抬升、不显示快捷入口；颜色不是状态的唯一表达方式。

### 列表与资源卡片

- 平台标准列表包括比较表格、资源列表、事件时间线和定义列表。
- 资源卡片包含状态边、资源图标、名称/版本、业务摘要、决策指标和固定操作区。
- 主操作使用图标加文字，次级操作使用图标按钮和 Tooltip，低频命令进入“更多”菜单。
- Hover 和 Focus 只改变边界、表面与操作可见性，不推动内容或改变卡片尺寸。
- 表格统一行高、表头、Hover、选中、操作列和局部横向滚动；Feature 继续拥有列与业务动作。

### 按钮和反馈

- 同一页面范围只保留一个视觉主按钮。
- Primary、Secondary、Ghost、Danger 均提供 Hover、Pressed、Focus、Loading 和 Disabled 状态。
- Loading 保持原宽高；危险操作必须同时表达对象、影响和后果，不能只依赖颜色。
- Inline Alert 使用中性表面、2px 语义边、图标、标题、正文和可选操作，不使用大面积语义色背景。
- Toast 只用于轻量、可恢复结果；持续错误和长任务在操作附近显示。

### 菜单和浮层

- Dropdown 用于上下文命令，Select 用于固定选项，Popover 用于筛选、日期和轻量编辑。
- 浮层从触发器方向展开，处理边缘碰撞并保留 8px 视口安全距离。
- 移动端复杂菜单或长选项使用底部 Drawer，不允许横向溢出。
- Dialog 只承载单一决策、确认和短表单；Drawer 承载复杂详情、审核和长表单；完整详情使用独立 URL。
- Dialog、Drawer 和菜单支持 Esc、焦点锁定、关闭后焦点归还与 Reduced Motion。
- Tooltip 只解释图标或陌生操作；Hover Card 只展示资源摘要；Artifact Preview 只展示文件和运行产物。

## 6. 核心页面

- 运行中心：紧凑步骤带与统计卡分层；总运行、运行中、通过率和异常运行使用共享统计卡；失败使用 Inline Alert；`FAILED` 与 `ERROR` 保持区分。
- 用户管理：全部、活跃、已禁用、需改密使用共享统计卡；保留超级管理员权限、URL 筛选、分页和 CRUD 浮层。
- 项目概览：项目状态、成员和测试资产使用共享统计卡；只展示业务事实和明确操作。
- Agent、用例、计划、环境、浏览器实例、评分器、实验、安全、审核和门禁通过共享 Token、列表、表格、按钮和浮层继承同一语言。
- 不改变路由、权限、API、分页、数据模型、业务状态、Trace、产物和审核逻辑。

## 7. 响应式与无障碍

- 验证 390px、1280px、1440px、1920px；禁止页面级横向滚动。
- 390px 下统计卡两列，表格使用列优先级、局部滚动或详情 Drawer。
- 所有交互具有键盘路径、可见焦点、可访问名称和正确语义，目标为 WCAG 2.2 AA。
- 使用 CSS 和 Radix 状态属性实现动效，不为装饰引入运行时动画框架。

## 8. 变更验证

每次影响设计系统或应用框架的变更至少验证：

1. Prettier、ESLint、TypeScript 和 Vitest。
2. 关键 Playwright 流程、Next.js 构建、架构门禁和 Bundle Budget。
3. 390px、1280px、1440px、1920px，以及浅色和深色。
4. 键盘焦点、对比度、抽屉、对话框、菜单碰撞和 Reduced Motion。
5. 路由、权限、API 契约、分页和专业测试字段无回归。
