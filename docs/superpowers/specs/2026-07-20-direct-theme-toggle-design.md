# 主题按钮单击直切设计规格

## 目标

将顶部主题控件从“点击后打开浅色/深色菜单”改为“点击图标立即切换主题”，减少一次选择操作，同时保持明确、克制和可访问的状态反馈。

## 方案比较

1. 保留双项菜单：状态表达完整，但二态选择仍要求两次点击，不符合本次目标。
2. 图标显示即将切换到的目标主题：动作提示直接，但用户难以从静态图标判断当前外观。
3. 图标显示当前主题，提示文本描述目标动作：当前状态与截图一致，单击路径最短，采用此方案。

## 交互契约

- 当前浅色时显示 Sun，单击后立即切换为深色并显示 Moon。
- 当前深色时显示 Moon，单击后立即切换为浅色并显示 Sun。
- 按钮的 `aria-label` 和共享 Tooltip 均描述下一步动作：`切换至深色` 或 `切换至浅色`；不叠加原生 `title`。
- 保留原有按钮尺寸、Hover、Focus、键盘 Enter/Space、`localStorage.theme` 持久化、根元素主题属性和跨标签页同步。
- 删除 Dropdown、RadioGroup、菜单项和选中勾选，不再创建主题浮层。
- 图标切换使用约 160-180ms 的轻量缩放/旋转入场；`prefers-reduced-motion: reduce` 下禁用该动画。

## 范围

- 修改共享 `ThemeToggle`、对应组件测试、工作区主题 Playwright 和必要的全局动效样式。
- 登录落地页与工作区继续复用同一个组件，因此两处同时获得直接切换行为。
- 账户中心的浅色/深色设置卡保持不变。
- 不修改根布局主题迁移、后端、OpenAPI、生成客户端、主题 Token 或其他菜单。

## 验收

- 组件测试证明单击一次即可在 `light` / `dark` 间往返切换，且页面不存在主题菜单角色。
- 组件测试证明按钮图标、可访问名称和 Tooltip 随当前主题同步更新。
- 历史 `system` 一次性迁移测试继续通过。
- Playwright 覆盖工作区单击直切、刷新持久化、无主题菜单和无 Hydration 错误。
- Web format、lint、typecheck、全量 Vitest、关键 Playwright、生产构建和局部截图检查通过。
