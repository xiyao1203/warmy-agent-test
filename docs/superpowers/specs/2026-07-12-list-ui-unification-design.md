# 全站列表 UI 统一设计

## 目标

统一 AgentTest 业务列表的列宽、长文本、操作区和编辑交互，确保 1280、1440、1920 像素桌面视口不出现横向滚动、内容遮挡或不可操作问题，并让窄屏仍能访问全部字段和操作。

## 覆盖范围

- 所有业务主列表：项目、Agent、用例集、测试计划、运行、浏览器实例、环境、模型配置、测试账号、用户、实验及相同层级的业务集合。
- 卡片式业务集合：操作入口统一使用相同的紧凑图标按钮和 Tooltip。
- 详情页只读子表：统一固定布局、长文本截断和无横向溢出，不改变 Trace、对比等专用交互。
- 项目名称编辑由行内表单改为 Dialog；创建、归档及后端契约不变。

## 组件设计

### 表格容器

共享 `Table` 默认使用 `width: 100%`、固定表格布局和裁剪容器。列宽由各 Feature 按信息优先级声明，操作列使用紧凑固定宽度。单元格必须允许内部内容收缩，不能通过 `min-width` 把操作区推出可视范围。

桌面端保留表头和全部列。窄屏允许单元格内容换行或纵向堆叠，但不隐藏字段、不生成横向滚动区域。

### 长文本

新增共享 `TruncatedText`：单行省略显示，鼠标悬停和键盘聚焦时展示完整内容；触发元素带可访问名称并可聚焦。项目名、ID、描述、URL、模型名、账号名和其他用户输入字段均使用该组件。状态、短日期和固定枚举不使用 Tooltip。

### 操作区

新增共享 `TableActionButton`：32x32 像素稳定尺寸，只显示 Lucide 图标；Tooltip 显示动作名称；`aria-label` 包含动作和资源名称。支持普通、危险、禁用、加载和 `asChild` 链接形态。`TableActions` 统一间距、居中和操作列宽度。

### 项目编辑

点击编辑图标打开受控 Dialog，输入框预填项目名。空名称不能提交；保存期间禁用关闭性操作并显示加载；失败信息保留在 Dialog 内；成功后关闭并清空状态。归档项目的编辑按钮保持禁用。

## 错误与无障碍

- Tooltip 同时响应 hover 和 `focus-within`，移动端仍可通过可访问名称理解图标。
- 图标均标记 `aria-hidden`，按钮本身提供完整 `aria-label`。
- 异步错误靠近对应 Dialog 或操作展示，不使用只靠颜色表达的状态。
- 不改变权限判断、请求参数、数据隔离和确认弹窗。

## 验证

- 共享组件测试覆盖固定布局、无横向滚动类、截断完整文本、Tooltip、图标按钮尺寸、危险和加载态。
- 项目列表组件测试覆盖弹窗编辑、长文本提示和纯图标操作。
- 现有各 Feature 测试用于防止 CRUD、链接、确认和权限行为回归。
- Playwright 在 1280、1440、1920 和 390 像素视口检查 `scrollWidth <= clientWidth`、操作可见、项目编辑 Dialog 和 Tooltip。
- 运行 format、lint、typecheck、Web 全量测试和 production build。
