# 全站资源列表自适应对齐与操作提示设计

## 背景与目标

项目、Agent、用例集、测试计划等资源列表当前使用固定百分比列宽。字段内容长度和操作按钮数量变化后，部分列出现大面积空白，操作列被压缩，Tooltip 文本竖排，表头和数据的视觉中心也不一致。

本次统一资源列表的内容驱动列宽、表头/数据对齐和图标操作提示，不改变 CRUD 流程、路由、权限、API 或业务状态。

## 范围

### 表格型资源列表

- 项目、Agent、用例集、用例详情、测试计划、测试执行、环境、浏览器实例、测试账号、用户、项目成员。
- 模型配置当前使用原生表格，迁移到相同的列宽和操作规范。
- 详情页中的 Trace 对比、实验结果对比等数据分析表保留其数值对比布局，不套用资源列表列宽。

### 卡片或行式资源列表

- 评分器、门禁、会话历史及其他带图标操作的资源集合统一补齐动作 Tooltip。
- 专业表单中的删除步骤、删除数据绑定等编辑器内部按钮不属于资源列表，不改变其布局。

## 共享组件设计

### 内容驱动表格

`Table` 默认使用自动布局。浏览器根据表头、单元格内容和语义最小宽度分配列宽；页面不再使用百分比宽度凑满 100%。表格容器保留横向滚动作为窄屏兜底，移动端已有卡片化布局继续生效。

`TableHead` 和 `TableCell` 默认居中。短值直接居中；名称、说明、资产摘要等多行内容使用共享 `TableValue` 容器，使内容块整体位于列中心，块内文字保持左对齐，避免多行文本难读。

列宽使用内容语义而不是页面百分比：标识/状态/日期列不换行并使用稳定最小宽度；名称/摘要列使用语义最小/最大宽度和 `TruncatedText`；操作列根据按钮数量自然扩展且禁止换行和挤压。

### 操作按钮

`TableActionButton` 区分两个名称：

- `label`：简短 Tooltip 文本，例如“进入”“编辑”“管理”“删除”。
- `accessibleLabel`：完整无障碍名称，例如“编辑项目 Mission Acceptance”。缺省时回退到 `label`。

所有图标型资源操作必须通过 `TableActionButton` 或同一 `Tooltip` 原语显示动作名称，并同时支持鼠标 hover 和键盘 focus。危险操作仍保留确认 Dialog，禁用、加载、权限判断和链接行为保持不变。

## Feature 迁移规则

每个列表只声明字段语义：内容列最小宽度、短值不换行、操作按钮名称。删除 `table-fixed` 和百分比列宽，不为单条测试数据写特殊分支。

多行动作统一放进 `TableActions`；单动作也使用相同按钮。原生表格迁移到共享 `Table`。卡片列表只补 Tooltip，不改变卡片结构。

## 响应式与可访问性

- 1280、1440、1920 桌面视口中，表头与对应数据块中心一致，操作 Tooltip 横向可读。
- 390 像素窄屏保留现有卡片化列表；无法卡片化的宽表允许容器内横向滚动，不让整页横向溢出。
- Tooltip 使用简短动作文本，不包含可能很长的资源名；完整资源名只用于 `aria-label`。
- 图标继续 `aria-hidden`，按钮保留焦点样式和至少 32x32 像素稳定尺寸。

## 验收

- 共享组件测试覆盖自动布局、默认居中、多行 `TableValue`、动作/无障碍双名称和 Tooltip。
- 各核心列表测试覆盖操作名称、链接/回调、删除确认和移动布局不回归。
- 架构测试扫描资源列表，禁止重新引入 `table-fixed`、百分比列宽和未带 Tooltip 的图标操作。
- Playwright 在 390、1280、1440、1920 视口验证项目、Agent、用例集、测试计划和运行列表的列中心、操作可见、Tooltip 可读及整页无横向溢出。
- 运行 Web format、lint、typecheck、全量 Vitest、production build、Bundle Budget 和完整 Playwright。

## 非目标

- 不修改产品字段、排序、筛选、分页、CRUD、权限、API、数据库或主题 Token。
- 不把资源列表改成卡片墙，不调整详情页对比表和专业表单编辑器。
- 不依据当前截图中的单条测试数据硬编码列宽。
