# 首屏与 Tooltip 瞬态稳定设计规格

## 目标

消除两类非必要的界面滞留和闪烁：主题按钮点击后 Tooltip 持续显示，以及落地页刷新检查会话时两个入口按钮短暂显示“正在检查”。

## 根因

- 共享 Tooltip 使用 `group-focus-within` 控制可见性。鼠标点击会让按钮保持焦点，即使指针移开，Tooltip 仍保持不透明。
- `LoginScreen` 把会话查询的 pending 状态直接映射为顶部与 Hero 按钮文案“正在检查”，因此每次硬刷新都会暴露内部认证过程。

## 方案比较

### Tooltip

1. 仅在主题按钮点击后主动 `blur()`：可快速隐藏，但破坏焦点管理，只修复单个调用点。
2. 完全删除焦点触发：鼠标问题消失，但键盘用户无法获得 Tooltip。
3. 共享 Tooltip 使用 Hover 或后代 `:focus-visible`：鼠标点击不滞留，键盘 Tab 仍可见，采用此方案。

### 落地页

1. 使用通用“进入平台”文案覆盖全部认证状态：稳定，但改变已确认的登录/工作台信息表达。
2. 在客户端持久化上次认证状态：可减少闪烁，但状态可能过期，不能作为认证事实。
3. pending 时保持未登录入口文案并禁用；确认已登录后才切换为工作台文案：改动最小、无内部状态文案，采用此方案。

## 行为契约

- Tooltip 在鼠标 Hover 时显示；鼠标点击控件后立即隐藏，即使指针仍停留在控件上也不滞留。
- Tooltip 在键盘 `:focus-visible` 时显示，保留 WCAG 键盘支持。
- 该规则在共享 Tooltip 实现，覆盖主题按钮、列表操作、侧栏和其他复用位置。
- 落地页会话查询 pending 时，顶部按钮保持“登录”，Hero 按钮保持“登录并开始”。
- pending 期间按钮保持禁用并添加 `aria-busy="true"`，不允许在身份未知时打开错误入口。
- 会话确认未登录后按钮启用且文案不变；确认已登录并解析入口后显示“工作台 / 进入工作台”。
- 为顶部按钮增加稳定最小宽度，避免“登录 / 工作台”切换造成明显水平位移。

## 同类审计边界

- 扫描所有 `isPending`、`isLoading` 和“正在/检查中”按钮文案。
- 用户点击提交后出现的“正在登录 / 保存中 / 修改中 / 正在执行”等是有效进度反馈，不属于首屏闪烁，继续保留。
- `/` 路由的“正在验证身份…”是整页路由守卫，不是可操作按钮，也不属于本任务。

## 验收

- 组件测试覆盖 Tooltip 新的 Hover/键盘触发类契约。
- 登录页组件测试证明 pending 时没有“正在检查”，稳定入口文案保持禁用并具有 `aria-busy`。
- Playwright 延迟会话接口，验证刷新首屏无“正在检查”；验证主题按钮点击、移开指针后 Tooltip 隐藏，键盘聚焦时仍显示。
- Web format、lint、typecheck、全量 Vitest、关键 Playwright、生产构建和截图检查通过。
