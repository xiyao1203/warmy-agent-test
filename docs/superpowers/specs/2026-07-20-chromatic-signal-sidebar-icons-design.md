# 工作台双色信号侧边栏图标设计规格

## 目标

在不改变工作台信息架构、路由、权限和密度的前提下，将当前单色 Lucide 导航升级为用户确认的 A「双色信号线」体系：图标有明确色彩、业务辨识度和轻量动效，折叠后仍通过 Hover 与键盘聚焦显示完整文本。

## 参考结论

- React Bits `Line Sidebar`：采用细状态线、局部位移和清晰层级，不引入大面积光效或循环动画。
- React Bits `Glass Icons`：只吸收“前后层次”概念，不使用玻璃方块载体，避免导航过重。
- Iconfont：用户提供的 `type=4` 是单色图标库；同时检查 `type=2` 多色图标库后，采用“细线主体 + 小面积第二信号色”，不复制第三方 SVG 源码。

## 视觉体系

- 继续使用 Lucide 的稳定语义几何，图标本体为 17px，载体保持 24px 且无可见方框、胶囊或底板。
- 每个入口绑定稳定语义色：珊瑚、蓝、青绿、琥珀、靛紫五组；同一业务类别保持一致映射。
- 第二颜色使用短斜线信号，不使用角标数字、圆点通知、渐变或发光球。
- 非激活图标保留颜色但降低强度；Hover/Focus 恢复完整强度并做 `translateX(1px) scale(1.04)`。
- 激活项继续使用 2px 左状态边，颜色与当前图标一致；背景只使用该颜色的低透明度混合，文本仍保持高对比。
- 浅色和深色主题分别定义语义 Token，不在组件内写原始颜色。

## 导航映射

| 色组 | 入口 |
|---|---|
| 珊瑚 | 项目列表、测试 Agent、人工审核 |
| 蓝 | 概览、浏览器实例、实验对比 |
| 青绿 | 智能体、测试执行、发布门禁 |
| 琥珀 | 环境与凭证、安全测试 |
| 靛紫 | 测试用例、测试计划、评分器、模型配置、用户与权限 |

图标语义沿用现有导航并对不够明确的入口做小幅修正，例如人工审核使用 `BadgeCheck`，避免普通勾选与审核流程混淆。

## 展开、折叠与移动端

- 桌面展开宽度保持 208px，折叠宽度保持 56px；行高 34px，不增加统计卡片或额外区域。
- 折叠时仅隐藏可见文字，不删除可访问名称；共享 Tooltip 在图标右侧显示入口文本，支持 Hover、`:focus-visible` 和 Escape。
- 展开时不显示冗余 Tooltip，文字截断规则保持现状。
- 移动端继续使用现有导航抽屉并复用同一图标和色彩映射，不创建第二套导航模型。

## 动效与无障碍

- 动效时长使用 `--motion-fast`，只响应 Hover、Focus 和选中状态，不循环、不弹跳。
- `prefers-reduced-motion: reduce` 下移除位移、缩放和斜线运动。
- 所有链接继续保留 `aria-label`、`aria-current`、可见焦点和权限门禁；颜色不是唯一状态表达，激活背景、状态边和字重同时存在。

## 影响范围

- 新增共享布局图标组件，导航数据增加语义色字段。
- 更新布局 CSS 与明暗主题 Token。
- 更新组件、Token 和关键 Playwright 测试。
- 无后端、数据库、API、OpenAPI、生成客户端、权限、依赖或运行配置变化。

## 验收标准

1. 所有桌面与移动项目导航使用 `chromatic-signal` 图标契约，不再标记为 `monochrome`。
2. 五组颜色在浅色和深色主题下清晰且克制，无方块底板、渐变、发光或图标重叠。
3. 折叠侧边栏 Hover/键盘聚焦显示文本，展开时显示正常标签。
4. 激活状态同时具备语义色状态边、低透明背景和 `aria-current`。
5. 390px 移动抽屉不横向溢出；Reduced Motion 无图标变换。
6. 前端 format、lint、typecheck、unit/component tests、关键 Playwright 和 build 通过。
