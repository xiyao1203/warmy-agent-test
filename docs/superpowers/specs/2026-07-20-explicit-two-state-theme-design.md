# 浅色 / 深色二态主题设计规格

## 目标

平台主题只允许用户显式选择浅色或深色。顶部外观菜单与账户偏好均不再展示“跟随系统”，操作系统主题变化不再改变平台外观。

## 行为契约

- 前端主题偏好类型收敛为 `light | dark`。
- 无有效本地偏好时，首次加载读取当前 `prefers-color-scheme`，选择对应浅色或深色并立即持久化；此后不再监听系统变化。
- 历史本地值 `system` 按相同方式一次性迁移为当时实际主题，并写回 `localStorage.theme`。
- 有效的 `light` 或 `dark` 在刷新、路由切换和系统主题变化后保持不变。
- 根元素 `class`、`data-theme`、`data-theme-preference` 与 `color-scheme` 始终使用同一个显式主题值。

## 界面

- 顶部外观按钮显示当前主题的 Sun/Moon 图标，单击后直接切换到另一主题，不打开选择菜单。
- 账户中心外观主题仅显示两项，布局由三列改为两列。
- 不改变按钮尺寸、菜单动效、焦点、明暗配色 Token 或其他页面布局。

## 兼容边界

- 后端与生成客户端继续接受历史 `system`，避免 API 契约变化；前端收到该值时映射为当前已解析主题，不向用户展示第三种模式。
- 不修改任何与系统管理、System Prompt、系统推断或其他业务语义相关的 `system` 字段。

## 验收

- 组件测试覆盖两项菜单、显式持久化、历史 `system` 迁移和无系统监听。
- 账户偏好测试证明只显示浅色/深色。
- Playwright 覆盖刷新持久化、历史迁移、系统主题变化不再跟随以及无水合错误。
- Web format、lint、typecheck、全量 Vitest、关键 Playwright 和生产构建通过。
