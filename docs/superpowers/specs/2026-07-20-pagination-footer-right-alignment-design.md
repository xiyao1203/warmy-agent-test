# 分页底栏右侧操作组设计规格

## 目标

统一所有资源列表的分页底栏：总数作为阅读信息保留在左侧，页大小、页码信息与翻页按钮作为一个连续操作组靠右，解决页大小选择器单独留在左侧造成的视觉割裂。

## 布局

- 桌面端单行显示：左侧 `共 N 条`；右侧依次为 `10 条/页`、`第 X / Y 页`、首页/上一页/页码/下一页/末页。
- 右侧操作使用一个稳定容器，整体 `margin-left: auto` 并右对齐。
- 390px 等窄屏允许右侧操作组内部换行，但每一行仍向右对齐；页大小选择器不得回到总数一侧。
- 保留现有 32px 翻页按钮、112px 页大小选择器、间距、焦点、Disabled 和明暗主题样式。

## 边界

- 只修改共享 `ResourcePagination` 及其测试和核心分页 E2E。
- 不修改 `page`、`page_size`、URL 同步、可选页大小、请求、API、数据结构或权限。
- 落地页不在本任务中修改。现有落地页继续遵守 `docs/superpowers/specs/2026-07-20-precision-motion-platform-ui-design.md`。

## 验收

- 组件测试证明总数与右侧控制分属两个语义分组，且页大小位于右侧分页操作组内。
- 桌面端右侧操作组右边缘与分页底栏内容边缘对齐。
- 390px 下无页面级横向滚动，右侧操作组换行后仍右对齐。
- Web format、lint、typecheck、相关 Vitest、核心分页 Playwright 和生产构建通过。
