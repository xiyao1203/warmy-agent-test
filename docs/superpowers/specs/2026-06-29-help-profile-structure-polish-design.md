# 帮助中心与个人资料体验二次优化设计

## 1. 背景与目标

上一轮已建立账户中心、帮助搜索和反馈提交能力，但页面仍存在结构割裂、重复外壳、卡片密度过高和部分占位式内容。此次在不改变后端契约、权限模型和全局品牌的前提下，明显调整信息架构，使帮助中心与个人资料页面更接近 Linear / Vercel 式克制、高密度研发工具体验。

通过条件：

- 帮助首页、教程、测试用例指南、快捷键和反馈页使用同一套帮助中心外壳。
- 个人资料以清晰的身份摘要、资料详情和账户元数据呈现，编辑态保持局部、可恢复且状态明确。
- 所有现有真实交互继续可用，不新增伪功能或无效入口。
- 桌面与窄屏均无关键内容遮挡、裁切或横向溢出。

## 2. 设计方向

采用“统一框架重构”方案。

- 保留现有语义设计 Token、Lucide 图标、字体和基础组件。
- 减少营销式渐变、彩色模块卡和重复大卡片，以分隔行、轻量面板和稳定层级组织内容。
- 页面标题使用现有工作台规范：约 24 像素、半粗体、紧凑说明文字。
- 交互保持完整，视觉调整不得把真实 API 状态替换为静态演示数据。

## 3. 帮助中心信息架构

### 3.1 统一外壳

新增可复用的帮助中心外壳，负责：

- 顶部栏：返回应用、帮助中心身份、反馈入口。
- 桌面左侧目录：首页、快速开始、测试用例、教程、快捷键、反馈。
- 主内容区：面包屑、页面标题、说明和页面内容。
- 窄屏：顶部保留返回和标题，目录转换为可横向滚动的分段导航。

各路由页面只组合内容，不再复制顶部栏和容器尺寸。

### 3.2 帮助首页

- 首屏由标题、说明和全宽搜索组成，不使用营销式大 Hero。
- 搜索结果直接在搜索区下方展示，并提供结果数量、清空和无结果反馈入口。
- “推荐路径”用紧凑列表呈现新用户从创建项目到查看报告的步骤。
- “浏览主题”按真实产品领域组织，使用统一单色图标和边框，不使用每卡不同渐变。
- FAQ 使用语义化 `details/summary` 或现有可访问折叠组件。
- 移除无对应内容的锚点、虚假联系方式和不可执行入口。

### 3.3 子页面

- 教程页不使用 emoji 伪缩略图；在无真实视频资源时改为课程目录式列表，并明确内容类型与预计时长。
- 测试用例指南使用文章式内容结构、步骤编号和最佳实践清单，减少重复卡片。
- 快捷键使用统一表格化分组，保留清晰的 `kbd` 语义。
- 反馈页复用统一外壳，表单保持真实提交、校验、提交中、失败重试和成功回执。

## 4. 个人资料信息架构

### 4.1 账户中心框架

- 账户中心保留资料、偏好、通知和安全四个分区。
- 桌面左侧导航使用轻量选中态和简短说明；移动端改为横向分段导航。
- 内容宽度收敛，标题、说明和分区内容对齐，避免宽屏下信息过度拉伸。
- 非法 `section` 查询值回退到资料分区，避免导航无选中状态。

### 4.2 资料页面

- 顶部使用紧凑身份摘要：首字母头像、名称、登录邮箱、角色和账号状态。
- 基本资料以分隔行展示，不再为每个字段单独套卡片。
- 编辑资料在基本资料区域内切换，不跳页；保存、取消、加载和错误状态均就地展示。
- 邮箱沿用当前 API 能力；若后端返回不可编辑错误，使用现有错误机制明确反馈，不伪造成功。
- 账户元数据展示用户 ID、角色与状态；长 ID 可换行或截断并保留完整可访问文本。

## 5. 组件边界与数据流

- `features/help` 提供帮助导航数据、统一外壳、搜索与反馈组件的公开出口。
- `features/account` 保持账户导航、资料、偏好、通知与安全组件边界。
- 路由页面仅传入标题、说明和内容，不承载搜索、资料更新或反馈提交规则。
- 现有 TanStack Query 查询键和 mutation API 保持不变，避免引入缓存回归。
- 不新增后端字段、API、数据库表或生成客户端变更。

## 6. 状态、无障碍与响应式

- 异步资料区保留结构化加载状态；失败状态提供可理解提示，已有重试能力时展示重试动作。
- 搜索、导航、表单和折叠内容支持键盘操作，图标按钮具有可访问名称。
- 当前导航使用 `aria-current="page"`，表单 Label 与错误信息建立关联。
- 1280 像素以上展示左侧导航；768–1279 像素压缩间距；小于 768 像素使用单列和横向分段导航。
- 所有动态过渡尊重 `prefers-reduced-motion`，不添加装饰性动画。

## 7. 测试与验收

先更新或补充失败测试，再实现结构调整。至少覆盖：

- 帮助外壳在各子路由的导航和当前页状态。
- 帮助搜索匹配、清空和无结果状态。
- 资料区加载、显示、进入编辑、取消、保存成功和保存失败。
- 非法账户分区参数回退。
- 反馈表单成功、失败和继续反馈。
- 关键控件的可访问名称和语义状态。

验证命令：

```text
pnpm --filter @warmy/web lint
pnpm --filter @warmy/web typecheck
pnpm --filter @warmy/web test -- <相关测试>
pnpm --filter @warmy/web build
```

人工检查桌面与窄屏的结构、焦点、裁切和横向溢出。用户已明确不使用浏览器视觉伴侣，因此本轮不把浏览器截图对比列为阻塞验收条件。

## 8. 非目标

- 不建设新的文档 CMS、全文搜索服务或教程视频资源。
- 不新增头像上传、两步验证、通知流或账号删除后端能力。
- 不调整全局侧栏、项目工作台或其他业务页面。
- 不修改全局品牌色、字体体系或语义设计 Token。
