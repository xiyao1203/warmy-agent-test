# Agent 对话克制增强版 UI 设计

## 1. 设计目标

在不改变现有 Agent 对话领域协议和平台导航的前提下，将页面收敛为“ChatGPT 式阅读体验 + Codex 式执行过程”。正文安静、宽松、可连续阅读；思考摘要、工具调用、确认和错误紧凑地嵌入同一条时间线；所有流式、取消、恢复和响应式状态保持真实可用。

## 2. 页面骨架

- 桌面端保留会话侧栏、主对话列和上下文区，但弱化三栏分割感。
- 会话侧栏默认宽度约 260px，背景与主画布仅通过轻微色差和细边界区分；搜索与新建入口固定在顶部。
- 主阅读列最大宽度 768px，正文、过程条目和输入框使用同一水平基线。
- 上下文区默认宽度约 288px，只展示关联资产和子 Agent 摘要，不与正文争夺视觉注意力。
- 1100px 以下隐藏上下文区并提供抽屉入口；760px 以下会话侧栏改为覆盖式抽屉，主列占满可用宽度。

## 3. 顶栏与会话侧栏

- 顶栏高度收敛到 48px，标题居中或随主阅读列对齐；侧栏、上下文和新建按钮使用一致的 36px 点击区域。
- 实时连接状态只在连接中、重连或离线时出现，正常状态不常驻占位。
- 会话条目使用 8px 圆角和单行标题；当前会话通过柔和底色与文字权重表达，不使用高饱和色块。
- 删除操作保持悬停出现，触屏设备通过条目菜单访问；删除确认不覆盖标题正文。
- 搜索为空、加载失败和无结果状态采用短句，不展示大卡片。

## 4. 消息与正文

- 用户消息靠右，最大宽度 76%，使用浅色胶囊块；长文本保持自然换行。
- 助手正文不使用外框或卡片，排版采用 15px 字号、约 1.75 行高和清晰的 Markdown 层级。
- 代码块、表格、引用和列表限制在阅读列内，横向溢出时仅内容区域滚动。
- 消息级操作在助手消息悬停或聚焦时出现，包含复制、重新生成和编辑相关入口；移动端始终提供可发现的更多按钮。
- 流式正文使用低干扰光标，不显示装饰性打字动画；完成后光标立即消失。

## 5. Codex 式过程时间线

- 思考摘要、工具准备、运行中、确认、完成、失败和停止共享一条左侧细线与统一图标尺寸。
- 默认只显示动词化标题和一行摘要，例如“正在调用执行 Agent”；详细参数、输出和错误通过可展开区域查看。
- 运行中使用轻量旋转图标和状态文字，不使用整块脉冲背景。
- 完成条目降低对比度；失败和等待确认保持足够强调，但颜色之外必须有图标和文字。
- 连续属于同一工具的开始、运行和完成状态在视觉上合并，避免同一任务出现三张重复卡片。
- “已停止”紧邻保留的部分回答，清楚表明该回答不完整。

## 6. 确认、错误与恢复

- 确认卡嵌入过程线，标题直接说明动作和风险；默认显示对象、能力和风险等级，参数详情可展开。
- 拒绝为次级按钮，确认执行为主按钮；提交时只锁定当前卡片并显示“正在执行”。
- 错误在相关过程条目内展示；仅无法归属的会话级错误使用输入区上方的窄提示条。
- 重连时保留正文草稿和当前过程状态，输入区上方显示一行“正在恢复实时连接”。
- 取消中按钮进入禁用状态并显示明确文案；收到服务端终止事件后再结束生成态。

## 7. 输入区与交互

- 输入区采用悬浮圆角容器，最大宽度与正文一致，背景与画布有轻微层次，聚焦时使用细边框和柔和阴影。
- 文本框从单行自动增长到最多约 8 行，超过后内部滚动；不因内容增长挤掉顶栏。
- 发送按钮位于容器右下方，无内容时禁用；生成中原位替换为停止按钮，避免布局跳动。
- Enter 发送、Shift+Enter 换行；快捷键信息放在可访问提示中，不塞进 placeholder。
- 空状态建议词改为轻量快捷入口，点击后填入输入框而非直接发送，给用户编辑机会。
- 输入区底部免责声明降低对比度，窄屏下允许换行但不占用过多高度。

## 8. 动效与无障碍

- 新增消息和过程条目仅使用 160–220ms 的淡入与轻微位移；重放历史时不逐条错峰动画。
- 遵守 `prefers-reduced-motion`，关闭非必要位移、旋转以外的装饰动画和全局平滑滚动。
- textarea、按钮、展开控件和抽屉具备可见焦点；textarea 也纳入全局焦点样式。
- `role=log` 只承载新增对话；连接、取消和错误使用独立 live region，避免重复朗读历史。
- 点击区域不小于 36px，关键移动端操作不依赖 hover。

## 9. 组件边界

- `TestAgentChat` 负责会话与生成编排，不继续堆叠纯展示逻辑。
- `ConversationTimeline` 负责有序消息与过程条目，并将工具状态归并为可展示模型。
- `ConfirmationCard` 负责单个确认状态、展开详情和操作反馈。
- `SessionList` 负责会话搜索、选择与删除交互。
- 新增上下文抽屉和移动侧栏时复用现有 Dialog/Sheet 基础组件，不创建平行弹层体系。

## 10. 验收标准

- 1440px、1024px 和 390px 三种视口下主阅读列、侧栏、上下文与输入区无裁切和横向页面滚动。
- 空会话、历史会话、流式正文、工具执行、等待确认、取消中、已停止、重连和错误状态视觉层级一致。
- 所有核心操作支持键盘，焦点可见；移动端可访问侧栏、上下文、消息操作和确认动作。
- 现有恢复、游标、取消和确认测试继续通过；新增时间线归并、输入区行为和响应式关键状态组件测试。
- TypeScript、定向 ESLint、组件测试和生产构建通过；浏览器完成桌面与窄屏视觉检查。
