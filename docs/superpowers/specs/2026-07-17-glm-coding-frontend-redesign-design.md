# GLM Coding 风格全前端重构设计规格

## 1. 背景与目标

现有平台功能完整，但 Apple 式大圆角、3D 彩色图标和分散的页面规则与高密度测试工作台不一致。本次重构参考 GLM Coding 的中性、克制、珊瑚红强调色与黑白工作区语言，建立平台自有设计系统。参考范围是视觉语法和交互密度，不复制对方商标、素材、文案或业务页面。

可验证目标：

1. 保留现有路由、权限、API、表单字段、业务状态和跨资源导航。
2. 使用单一设计 Token 层驱动浅色、深色和跟随系统三种主题。
3. 重构应用框架、公共原语和核心工作区，使全部业务模块一致。
4. 在 390/1280/1440/1920 像素下保持主要操作可用，满足 WCAG 2.2 AA 的基本键盘、焦点与对比度要求。

## 2. 产品原则

- **专业而克制**：中性表面与边界组织信息，不使用装饰性大渐变、厚重阴影、浮动卡片或多色导航图标。
- **状态可读**：颜色不是唯一状态载体，同时使用文字、图标或形状。
- **操作就近**：页面主操作放在标题右侧，筛选和批量操作与数据表相邻，不依赖 Toast 传达长任务。
- **工作区优先**：页面空间优先给对话、表格、Trace、结果与配置，框架本身保持紧凑。
- **渐进适配**：宽屏多栏，窄屏变成可访问的抽屉或分页，表格使用列优先级与局部滚动。

## 3. 信息架构

左侧导航分组，项目上下文不变：

- 工作台：测试 Agent、概览。
- 测试资产：智能体、测试用例、测试计划。
- 执行中心：测试执行、环境与凭证、浏览器实例。
- 评测与治理：评分器、实验对比、人工审核、安全测试、发布门禁。
- 系统设置：模型配置、用户与权限（仅超级管理员）。

顶部工具栏包含项目切换、面包屑、全局搜索/命令面板（`Cmd/Ctrl+K`）、快速创建、运行状态、帮助、通知、主题和账户。窄屏使用菜单按钮打开导航抽屉。

## 4. 视觉系统

### 4.1 颜色

浅色主题：

- 页面画布 `#f7f7f8`，弱表面 `#f1f1f3`，主表面 `#ffffff`。
- 主文字 `#18181b`，次文字 `#3f3f46`，弱文字 `#71717a`，边界 `#e4e4e7`。
- 产品主色使用珊瑚红 `#e94b43`，Hover/Active 使用更深阶，选中表面使用低透明红。

深色主题：

- 页面画布 `#111113`，弱表面 `#18181b`，主表面 `#1f1f22`，抬升表面 `#27272a`。
- 主文字 `#fafafa`，次文字 `#d4d4d8`，弱文字 `#a1a1aa`，边界 `#303036`。
- 主色提亮为 `#ff6b61`，保持深色表面上的对比度。

成功、警告、失败、信息只用于语义状态；业务页不得写原始色值。

### 4.2 排版、间距与层次

- 中文优先系统无衬线字体，数字使用 tabular nums，日志与代码使用等宽字体。
- 受控字号为 12/13/14/16/20/24/32px，页面标题通常为 20–24px。
- 4px 基础网格，控件高度 32/36/40px，页面标准间距 12/16/20/24px。
- 圆角 6/8/12px，胶囊只用于徽标与身份；阴影只用于 Dropdown、Popover、Drawer、Dialog、Toast。
- 快速动效 120ms，标准动效 180ms，支持 `prefers-reduced-motion`。

## 5. 页面模板

### 5.1 Agent 工作区

宽屏使用“会话历史 / 对话主区 / 计划与证据”三栏。对话消息还原为低装饰内容流，计划、执行、确认、结果使用专用卡片。右侧结构化配置与对话互通，重要成本/风险始终可见。

### 5.2 列表与详情

列表由紧凑页头、筛选工具条、表格、批量操作和分页组成。行引用、状态、质量、成本、风险和活动摘要保留。快速预览使用右抽屉，完整详情保留独立 URL。

### 5.3 运行与结果

顶部展示状态、进度、通过/失败/错误/审核中、成本与剩余时间；主区由结果列表、用例输入/输出/Trace/产物、评分/失败分类/审核意见三部分组成。`FAILED` 与 `ERROR` 始终具有不同文字标签。

## 6. 主题行为

主题选择器提供“浅色 / 深色 / 跟随系统”。选择写入 `localStorage.theme`，`system` 保持为明确值。跟随系统时监听 `prefers-color-scheme` 变化；首次渲染前内联脚本应用解析后主题，防止闪烁。`color-scheme` 与有效主题同步。

## 7. 无障碍、响应式与性能

- 全局搜索、主题选择、导航、抽屉和对话框使用可访问原语；图标按钮有名称与可见焦点。
- 390px 下主导航移入抽屉，顶部保留项目与主操作；多栏工作区按任务优先级折叠。
- 全站不产生页面级水平滚动；数据表本身允许局部滚动。
- 重型图表、Trace、画布和媒体按需加载；新增首屏 JavaScript 不得使全站 Bundle Budget 超限。

## 8. 验收标准

1. 已有页面和权限测试继续通过，路由与业务不回归。
2. 主题三态持久化、系统变化跟随、无首屏错色闪烁。
3. 导航分组、全局命令面板、快速创建、面包屑、收起/移动行为可键盘操作。
4. 登录、项目、测试 Agent、测试用例、计划、运行、结果与治理模块共用新 Token 和公共原语。
5. 通过格式化、Lint、TypeScript、Vitest、Playwright、Build、架构与 Bundle Budget 验证。
