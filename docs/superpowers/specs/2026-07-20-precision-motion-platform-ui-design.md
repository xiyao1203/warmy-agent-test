# Precision Motion 全平台 UI 与交互升级设计规格

## 1. 背景与目标

现有 Web 已具备 GLM Coding 风格的中性 Token、明暗主题、应用框架、共享表格和标准分页，但落地页仍采用营销式左右分栏与大卡片，侧边栏偏宽，统计摘要、列表卡片、操作按钮和浮层缺少统一的动态层次。部分页面还展示了设计取舍或实现范围等内部说明，不属于用户需要理解的业务内容。

本任务建立平台自己的 **Precision Motion** 语言：成熟、精密、时尚、紧凑，动效服务于状态和结构，不使用常见的 AI 产品装饰套路。

可验证目标：

1. 落地页首屏以 `Warmy Agent Test` 为产品名，使用真实平台工作流场景承载主视觉。
2. 登录成功关闭弹窗并停留落地页，主按钮更新为“进入工作台”；只有用户再次点击才进入后台。
3. 应用侧栏收敛为 208px/56px 的纯导航，不展示统计、数量、通过率或运行数据。
4. 建立统一的紧凑统计卡、资源列表、操作按钮、菜单、Dialog、Drawer、Tooltip、Hover Card、提示和 Toast 规则。
5. 运行中心、用户管理和项目概览直接迁移到新组件；其他业务页通过共享 Token、表格、列表与浮层原语同步收敛。
6. 保留现有路由、权限、API、分页、数据模型和业务行为。

## 2. 参考与原创边界

- Uiverse：参考 Hover、Pressed、Loading、菜单和按钮的触感反馈。
- Aceternity UI：参考边界响应、表面层次和内容进入的空间感。
- React Bits：参考数字更新、状态切换、列表变化和图标语义动画。

不直接复制第三方页面、商标、文案或完整组件库，不引入大面积渐变、霓虹、玻璃、光球、星芒、抽象 AI 图标或无意义循环动画。生产图标统一使用仓库现有 `lucide-react`。

## 3. 视觉基础

### 3.1 颜色与表面

- 沿用中性灰画布和珊瑚红品牌主色。
- 品牌色只用于主操作、焦点、选中和少量关键状态。
- 成功、警告、失败、信息使用语义 Token，并同时提供图标或文字。
- 普通页面分区保持无框或使用分隔线；卡片仅用于重复资源、指标、模态和真实工具表面。
- 阴影只用于 Hover 抬升、Dropdown、Popover、Dialog、Drawer 和 Toast。

### 3.2 尺寸与排版

- 圆角以 6px/8px 为主，普通卡片最大 8px。
- 控件高度为 32px/36px/40px；紧凑工作区优先使用 32px 或 36px。
- 页面标题保持 20px-24px，落地页产品名可使用 40px-58px。
- 字距固定为 0；数字使用 `tabular-nums`。
- 图标光学尺寸 17px、线宽约 1.75、承载区域 24px；不使用 Emoji、多套图标或临时手绘符号。

### 3.3 动效 Token

- 微交互：120ms-160ms。
- 菜单与轻浮层：140ms-180ms。
- 状态切换：180ms。
- Dialog：220ms。
- Drawer：260ms。
- 空间变化：不超过 280ms。
- 除运行中状态外不使用循环动画。
- `prefers-reduced-motion` 下关闭非必要动画，保留颜色、文字和结构状态。

## 4. 落地页与登录

### 4.1 首屏

- Header 保留品牌、主题和登录/工作台入口。
- H1 必须是 `Warmy Agent Test`，业务价值放在支持文案中。
- 首屏使用单列产品叙事，不使用左右“文案 + 卡片”结构。
- 主视觉是全宽、真实的测试运行界面，展示资源版本、运行状态、评分、审核和门禁证据。
- 首屏在桌面和移动视口均露出下一段内容。
- 未登录主操作为“登录”；已登录或登录成功后的主操作为“进入工作台”。
- 次操作滚动到产品证据场景，不打开登录弹窗。

### 4.2 后续内容

- 使用全宽内容带展示测试资产、执行、评估、审核和放行之间的真实业务关系。
- 使用运行记录、状态边、结果数字和真实平台界面切片，不使用虚构客户 Logo、价格、公开注册、虚假成功数据或营销式 Bento 卡片。
- Footer 只说明组织账号访问方式，不展示内部设计或开发说明。

### 4.3 登录行为

- 登录表单使用短 Dialog；标题、说明、错误和安全信息保持业务语言。
- 登录成功只关闭 Dialog 并更新本地工作台入口，不自动导航。
- 用户再次点击“进入工作台”时才导航到 returnTo 或首个项目测试 Agent。
- 已登录用户访问 `/` 或 `/login` 仍保持落地页。

## 5. 应用框架与导航

- 展开侧栏宽度 208px，收起宽度 56px，导航行高 34px。
- 侧栏只承载导航，不放数量、通过率、运行状态或其他统计。
- 当前项使用 2px 短状态边、轻背景和文字强调。
- 收起状态使用 Tooltip 展示名称，不依赖原生 `title`。
- 顶部栏只保留项目切换、全局搜索、快速创建、运行入口、主题、帮助、通知和用户菜单。
- 移动端导航进入可访问 Drawer；桌面收起状态持久化在现有 `sidebar-collapsed` 偏好中。
- 不改变权限控制、项目切换和路由语义。

## 6. 组件系统

### 6.1 紧凑统计卡

采用确认的 **C：图标结合状态边**：

- 桌面高度约 88px，移动端约 84px；桌面四列，移动端两列。
- 17px Lucide 图标与 2px 状态边组成识别标记，不使用彩色方形或圆形图标底座。
- Hover 时状态边展开、图标轻移、顶部短高光出现，快捷入口在固定区域内显现。
- 运行中图标允许持续旋转；完成描线、数字更新和异常提醒只执行一次。
- 卡片支持默认、Hover、Focus、选中、Loading 骨架、空数据、错误、无权限、不可点击和明暗主题。
- 不可点击卡片不抬升、不显示箭头；操作出现不能改变卡片尺寸。

### 6.2 列表与资源卡片

平台维护四类标准列表：比较表格、资源列表、事件时间线和定义列表。

资源卡片包含状态边、资源图标、名称/版本、业务摘要、决策指标和固定操作区。主操作使用图标加文字，次级操作使用图标按钮和 Tooltip，低频命令进入“更多”菜单。Hover 和 Focus 只改变边界、表面与操作可见性，不推动内容。

表格保留 Feature 对列和业务动作的所有权，统一行高、表头、Hover、选中、操作列和局部横向滚动，不构建配置驱动的万能 DataTable。

### 6.3 按钮

- 同一页面范围只保留一个视觉主按钮。
- Primary、Secondary、Ghost、Danger 提供 Hover、Pressed、Focus、Loading、Disabled 状态。
- Loading 保留原宽高；危险操作不能只通过颜色表达。
- 熟悉的工具动作使用 Lucide 图标；陌生图标必须有 Tooltip 和可访问名称。
- 胶囊仅用于状态、筛选标签和身份，不用于普通命令按钮。

### 6.4 下拉菜单与浮层

- Dropdown 用于上下文命令；Select 用于固定选项；Popover 用于筛选、日期和轻量编辑。
- Dropdown 从触发器方向展开，启用边缘碰撞和 8px 视口安全距离。
- 移动端复杂菜单或长选项使用底部 Drawer，不横向溢出。
- Dialog 只承载单一决策、确认和短表单；Drawer 承载复杂详情、审核和长表单；完整详情保留独立 URL。
- Dialog、Drawer、Dropdown 支持 Esc、焦点锁定、关闭后焦点归还和 Reduced Motion。

### 6.5 提示、预览与反馈

- Tooltip 只解释图标或陌生操作。
- Hover Card 只展示资源摘要，不承载必需操作。
- Artifact Preview 展示文件、图片、视频、日志或运行产物，不与 Hover Card 混用。
- Inline Alert 使用白色/中性表面、2px 语义状态边、图标、标题、正文和可选操作，不使用大面积语义色背景。
- Toast 只用于轻量、可恢复结果；持久错误和长任务必须就近显示。
- 危险操作必须明确对象、影响、权限和后果，并提供二次确认与结果反馈。

## 7. 核心页面迁移

### 7.1 运行中心

- 保留页面标题、测试计划版本选择、启动执行、筛选、分页和列表契约。
- 工作流说明收敛为紧凑步骤带，不与统计卡竞争视觉层级。
- 总运行、运行中、通过率、异常运行使用新统计卡。
- 请求失败使用 Inline Alert；运行列表继续区分 FAILED 与 ERROR。

### 7.2 用户管理

- 全部用户、活跃、已禁用、需改密使用新统计卡。
- 保留超级管理员权限、筛选 URL、分页和现有 CRUD Dialog/Drawer。
- 无权限、服务错误和 Loading 保持独立状态。

### 7.3 项目概览

- 项目状态、成员和测试资产使用紧凑统计卡语言。
- 删除“当前页面只展示什么”等内部范围说明，替换为业务事实和明确操作。
- 成员表格和运行入口保持原权限与路由。

### 7.4 其他工作区

- Agent、用例、计划、环境、浏览器实例、评分器、实验、安全、审核和门禁通过共享 Token、Button、ListCard、Table、Dropdown、Dialog、Drawer、Tooltip 和反馈组件继承新语言。
- 测试 Agent 和结果工作台保持现有信息架构，只收敛表面、边界、按钮和状态动效，不改变对话、Trace、产物和审核逻辑。

## 8. 文案规则

产品界面只展示：当前业务事实、用户可执行的下一步、必须了解的约束/风险/错误。

禁止展示：设计思考、技术实现、当前版本取舍、页面故意不展示的内容、开发范围、空泛的“AI 驱动/智能赋能/重新定义”文案。必须删除现有“减少用户理解成本”“不展示桌面壳”“当前概览页只展示摘要”和“尚未接入的视频入口”等内部说明。

## 9. 响应式、无障碍与性能

- 验证 390px、1280px、1440px、1920px；页面级禁止横向滚动。
- 390px 下统计卡两列，表格使用列优先级、局部滚动或详情 Drawer。
- 所有交互支持键盘、可见焦点、可访问名称和正确语义；目标为 WCAG 2.2 AA。
- 明色、暗色、跟随系统三态保持等价功能。
- 动效使用 CSS 和 Radix 状态属性优先，不为装饰引入运行时动画框架。
- 不新增第三方运行时依赖；若后续确需引入，必须另行完成版本、许可证、安全、体积和回滚评估。

## 10. 架构与数据边界

- 新共享组件位于 `apps/web/src/components/ui`，不得依赖 Feature。
- Feature 继续通过公开 `index.ts` 依赖，不新增跨 Feature 内部导入。
- 页面只组合 Feature，不新增业务规则。
- 不修改 Control API、数据库、OpenAPI、生成客户端、权限策略、分页和业务状态。
- 不可信 Markdown、HTML、URL 和产物继续沿用现有安全边界。

## 11. 验收标准

1. 落地页首屏、登录成功停留和二次进入工作台行为由组件测试和 Playwright 覆盖。
2. 侧栏 208px/56px、34px 行高、纯导航、折叠 Tooltip 和移动 Drawer 通过测试。
3. 统计卡全部状态、图标语义动效、固定动作区域和 Reduced Motion 通过组件测试。
4. Dropdown 碰撞、Dialog/Drawer 动画与焦点、Tooltip/Hover/Alert 边界通过组件测试。
5. 运行中心、用户管理、项目概览和核心列表在 390/1280/1440/1920 与双主题下无重叠、截断或布局跳动。
6. 产品界面不再出现内部设计/实现说明。
7. Web format、lint、typecheck、全量 Vitest、关键 Playwright、Build、Bundle Budget 和前端架构门禁通过。
