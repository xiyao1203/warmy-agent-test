# Agent 资产与平台功能完整闭环设计

## 1. 目标

将现有“Agent 基本登记 + 版本列表”升级为可验证的完整资产闭环：用户可以完成 Agent 登记、连接配置、连接验证、发布、当前/基线设置，并从 Agent 详情追踪它在测试计划、运行、产物、实验、安全扫描和发布门禁中的真实使用情况。

## 2. 产品与信息架构

### 2.1 核心概念

- `Agent` 是稳定身份，保存项目、名称、类型和描述。
- `AgentVersion` 是不可变执行快照，保存连接协议、请求/响应映射、限制、模型与工具元数据、凭证绑定和插件信息。
- 测试计划、对话调试、安全扫描和运行只引用确切的已发布 `AgentVersion`，不隐式解析“最新版本”。
- `current_version_id` 用于默认选择和日常使用；`baseline_version_id` 用于版本对比建议，不替代测试计划中的显式版本引用。

### 2.2 列表与创建

- 列表主操作由“查看”改为“管理”，Agent 名称与整行均可进入详情。
- 列表展示接入类型、当前版本、基线版本、发布状态和更新时间。
- 创建 Agent 只登记稳定身份；创建成功后立即进入详情概览。
- 无版本时，概览展示接入步骤：基本信息 → 创建连接版本 → 测试连接 → 发布 → 设为当前。
- 删除使用确认对话框；存在任何版本或下游引用时后端拒绝删除并返回引用摘要。

### 2.3 详情页

- 默认 Tab 为“概览”，不是“版本历史”。
- 概览展示身份信息、接入健康度、当前/基线版本、最近运行和关联资产计数。
- 配置 Tab 展示当前版本的完整配置，并提供编辑 Agent 基本信息入口。
- 版本 Tab 支持草稿创建/编辑、连接测试、发布、设为当前、设为基线和两版本对比。
- 运行、产物、实验、安全和门禁使用真实关联数据；空态说明下一步操作，不使用占位说明代替功能。

## 3. 后端设计

### 3.1 Agent 命令

- 增加删除 Agent 命令：只有无版本、无测试计划引用、无运行/安全等引用时允许删除。
- 增加设置当前版本、设置基线版本应用命令，经仓储持久化并记录审计。
- 仅允许将属于同一 Agent 且已发布的版本设为当前或基线。
- 发布首个版本后可自动设为当前版本；后续发布不自动覆盖用户选择。

### 3.2 版本配置

在现有配置基础上完整支持：

- API/Web 地址、调用协议、请求模板、响应提取路径。
- 认证凭证绑定、Header/变量注入由环境与凭证模块提供选择。
- 模型、模型参数、System Prompt、工具清单与 Schema。
- 代码版本、Git Commit、知识库/数据版本。
- 超时、最大步骤、成本限制。
- Adapter/插件 ID 与版本。

所有字段进入 `AgentConfig`、API Schema、版本详情、Diff 与运行快照。敏感凭证只保存绑定 ID，不回显明文。

### 3.3 关联摘要

新增项目隔离的 Agent 关联摘要查询，返回：

- 使用该 Agent 各版本的测试计划版本。
- 运行及其状态、通过率、时间和版本。
- 运行产物数量与最近产物。
- 以这些运行为输入的实验。
- 指向这些版本的安全扫描。
- 指向相关运行/实验的发布门禁。

查询必须按 `project_id` 校验；只返回详情页需要的摘要字段，完整详情继续使用各模块公开 API。

## 4. 跨模块数据流

1. 用户创建 Agent 和草稿版本。
2. 连接测试通过后发布版本，首个发布版本成为当前版本。
3. 测试计划只能选择同项目已发布版本，并保存确切 `agent_version_id`。
4. 发布测试计划时校验 Agent 版本、数据集版本和评分配置。
5. Run Source 从已发布测试计划加载 Agent 版本，生成不可变 `plugin_snapshot`。
6. Worker 按快照执行，Run 保存 Agent 版本、Trace、评分和产物关联。
7. 实验比较 Run；安全扫描直接引用 Agent 版本；门禁引用 Run 或 Experiment。
8. Agent 详情通过关联摘要回看上述资产，提供到原详情页的链接。

## 5. API 契约

- `PATCH /projects/{project_id}/agents/{agent_id}`：编辑名称和描述。
- `DELETE /projects/{project_id}/agents/{agent_id}`：无引用时删除，否则 `409` 返回引用摘要。
- `PATCH /projects/{project_id}/agents/{agent_id}/current-version`：设置已发布当前版本并持久化。
- `PATCH /projects/{project_id}/agents/{agent_id}/baseline-version`：设置已发布基线版本并持久化。
- `GET /projects/{project_id}/agents/{agent_id}/versions/{v1}/diff/{v2}`：同 Agent 版本差异。
- `GET /projects/{project_id}/agents/{agent_id}/relationships`：关联摘要。

所有写操作要求 CSRF、项目编辑权限和审计；所有资源检查 Agent、版本和引用的项目归属。

## 6. 错误与边界

- 草稿不能设为当前或基线。
- 已发布版本不能编辑。
- 不允许跨 Agent、跨项目设置版本或建立引用。
- 删除有版本或引用的 Agent 返回可读阻塞原因，不静默级联删除。
- 连接测试失败不阻止保存草稿，但发布前必须有最近一次成功验证记录；验证记录与版本配置摘要绑定，配置改变后失效。
- 关联模块不可用时详情局部显示错误与重试，不让整个 Agent 页面失效。

## 7. 测试与验收

### 后端

- Agent CRUD、项目隔离和权限。
- 草稿/发布不可变规则。
- 当前/基线只接受同 Agent 已发布版本且刷新后可恢复。
- 删除引用保护。
- 连接验证状态与配置变更失效。
- 关联摘要跨测试计划、Run、产物、实验、安全和门禁。
- Run 快照包含完整 Agent 调用与元数据配置。

### 前端

- 创建后进入详情；详情默认概览。
- 无版本接入步骤和完整版本表单。
- 编辑基本信息、删除错误、连接验证、发布、当前/基线和 Diff。
- 测试计划只列出已发布 Agent 版本。
- 运行/产物/实验/安全/门禁 Tab 展示真实数据与链接。
- 键盘操作、焦点、Tab 语义、唯一可访问名称和窄屏布局。

### 质量门禁

- 后端 format、Ruff、mypy 变更域、单元/契约/集成/架构测试。
- 前端 Prettier、ESLint、TypeScript、组件测试、关键 E2E、生产构建。
- OpenAPI 导出和生成客户端 drift 通过。

## 8. 非目标

- 不引入新的 Agent 类型或插件市场。
- 不改变既有 Worker/Temporal 执行协议，只扩充快照字段。
- 不自动删除历史版本或下游运行数据。
- 不将当前/基线指针替代测试计划中的显式版本选择。
