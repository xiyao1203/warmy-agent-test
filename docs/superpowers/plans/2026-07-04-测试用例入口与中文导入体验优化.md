# 2026-07-04 测试用例入口与中文导入体验优化

## 目标

降低测试用例管理的理解成本：用户进入“测试用例”后能直接看到新增用例入口，导入模板默认使用中文字段名，同时保留英文旧格式兼容；页面文案明确“用例集包含用例，用例发布后供测试计划引用”。

## 范围

- 前端测试用例/数据集列表和详情页的按钮、空态、辅助文案与操作优先级。
- 前端导入模板工具和导入向导模板下载内容。
- 后端数据集导入解析对中文字段名和中文枚举值的兼容。
- 前后端相关测试和任务记录。

## 不做

- 不新增数据库迁移。
- 不改变测试计划创建 API。
- 不引入新的资产类型；继续沿用“数据集版本承载测试用例”的现有模型。
- 不实现失败运行一键生成用例、复制归档等未完成的扩展入口。

## 实施步骤

1. 梳理现有测试用例新增、导入和创建测试计划入口，找出用户误解来源。
2. 把草稿版本详情页主操作调整为“新增用例”，导入作为并列操作，测试计划入口改成明确的下一步动作。
3. 将列表页面向用户的“创建数据集”弱化为“创建用例集”，并用简短文案说明用例集和用例列表的关系。
4. 导入模板默认输出中文表头/字段名，并在后端导入解析中兼容中文表头、中文 JSON key 和常用中文枚举值。
5. 更新前端组件测试和后端导入契约测试，运行格式、Lint、类型与相关测试。

## 验收

- 测试用例详情页草稿版本明确显示“新增用例”，用户不需要把“新建测试计划”误认为新增用例。
- 导入模板字段名默认是中文，导入仍兼容现有英文格式。
- 中文 CSV/JSON 导入可以预览成平台标准用例字段。
- 页面文案能说明“先创建用例集/新增用例/发布版本，再用这些用例创建测试计划”的链路。
