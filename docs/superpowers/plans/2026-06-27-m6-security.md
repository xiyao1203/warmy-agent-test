# M6：安全与门禁 Implementation Plan

> **Goal:** 建立 Promptfoo 安全扫描、安全策略引擎和 CI 质量门禁。

**Architecture:** Promptfoo 作为独立 npm 包接入，Policy Engine 在控制面扩展，CI 增强已有 GitHub Actions。

---

## Task 1: Promptfoo 集成

- [ ] 安装 `promptfoo` npm 包
- [ ] 配置 Prompt Injection 检测规则
- [ ] 配置敏感信息泄露检测规则
- [ ] 配置基础越狱（jailbreak）扫描规则
- [ ] 编写 CLI 入口：`pnpm --filter @warmy/web promptfoo:eval`
- [ ] 输出 JSON 报告到 `artifacts/promptfoo/`

**文件范围：**
- `apps/web/promptfooconfig.yaml`
- `apps/web/tests/promptfoo/`（检测用例）
- `apps/web/package.json`（新增 script）

## Task 2: 安全策略引擎

- [ ] 创建 `apps/control-api/src/agenttest/modules/security/` 模块
- [ ] 领域模型：SecurityPolicy、PolicyRule、PolicyViolation
- [ ] 应用层：PolicyEngine（检查请求、工具调用、步骤限制）
- [ ] API 端点：策略 CRUD + 运行检测
- [ ] 迁移：security_policies 表

**策略规则：**
- 最大步骤数限制（默认 20）
- 单次执行超时（默认 5min）
- 禁止工具列表（可配置）
- 危险操作需确认

**文件范围：**
- `apps/control-api/src/agenttest/modules/security/`（domain/application/infrastructure/api）
- `apps/control-api/migrations/versions/0005_security_policy.py`
