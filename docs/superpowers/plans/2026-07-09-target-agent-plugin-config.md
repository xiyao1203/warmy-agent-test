# TASK-20260709-006：目标 Agent 插件可视化配置中心

## 背景

用户希望后续测试目标 Agent 时，不再通过写代码集成插件，而是在平台上可视化填写目标产品访问地址、账号密码或登录态、插件类型和安全边界。平台需要能把这些配置保存成 Agent 版本快照，并复用现有项目凭证、浏览器实例和运行链路。

## 目标

1. Agent 版本配置支持保存目标插件可视化配置快照。
2. 前端 Agent 版本弹窗提供“目标接入”配置页，支持插件模板、目标地址、登录方式、凭证绑定、浏览器实例、页面选择器和只读安全边界。
3. 用户可从配置页创建加密项目凭证，并在版本配置中只保存凭证 ID，不保存密码明文。
4. 配置能兼容现有 Worker 运行字段，尤其是 Web/Canvas 目标地址。
5. 补充测试覆盖安全约束和核心交互。

## 非目标

- 本任务不实现第三方插件市场安装和沙箱执行。
- 本任务不把测试账号旧接口改造成完整密钥系统。
- 本任务不绕过目标站账号额度、验证码、二次确认等外部限制。

## 实施步骤

1. 后端 AgentConfig 扩展 `target_config` JSON 快照字段，并补充序列化、反序列化和单元测试。
2. 前端扩展 Agent 版本配置类型，新增目标插件配置模型与默认模板。
3. Agent 版本弹窗改造成更清晰的分区：目标接入、请求映射、安全边界、元数据。
4. 在目标接入页加载项目凭证和浏览器实例，支持一键保存账号密码为加密项目凭证，并只把凭证 ID 写入版本配置。
5. 保存版本时自动同步 `api_url`、`web_url`、`plugin_id`、`plugin_version`、`credential_binding_ids` 和 `target_config`，兼容现有运行链路。
6. 增加前端组件测试，验证 URL+账号密码配置不会把密码写入 Agent 版本 payload。
7. 运行相关前后端测试、类型检查和构建；根据结果修复。

## 安全约束

- 密码、Token、Cookie 不允许写入 Agent 版本 `config`、测试用例、日志或 URL。
- 版本配置只保存 `credential_binding_id`、掩码提示和登录策略。
- 自动运行默认只读边界，禁止发布、删除、付款、权限变更和确认类高风险动作。
- Worker 后续取用凭证时必须通过授权接口获取短期最小凭证。

## 验收

- 创建 Agent 版本时可以选择“通用 Web Agent”或“TapNow 画布 Agent”模板。
- 填写目标访问地址后可保存版本，版本配置包含 `target_config.entry_url` 与兼容字段。
- 填写用户名密码并创建凭证后，保存 payload 不包含明文密码，只包含凭证 ID。
- 组件测试和 AgentConfig 单元测试通过。
