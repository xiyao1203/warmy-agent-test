# TASK-20260708-004：iconfont 视觉资产与 3D Logo 升级计划

## 背景

用户希望继续参考 `https://www.iconfont.cn/`，分别查看 3D 插画库、动效库和字体库，并应用到平台中，同时为平台设计一个 3D 插画风 Logo。

已在 iconfont 官网确认：

- 首页素材分类包含图标库、矢量插画库、3D 插画库、Lottie 库和字体库。
- 3D 插画库包含“3D缺省页”“B端蓝白科技”等适合 B 端平台的方向。
- Lottie 库包含小图标、环境类、即时创造等轻量动效方向。
- 字体库提供独立入口，适合作为标题、数字和品牌层级的参考。

本任务不直接下载或嵌入未知授权素材；采用官网分类和风格作为参考，落成本地原创 SVG/CSS 资产，避免登录态、外链稳定性和版权风险。

## 实施范围

1. 品牌 Logo
   - 新增 `BrandMark` 组件。
   - 将顶部品牌从纯文字升级为 3D 插画风 Logo + `Agent Test` 文案。
   - 收起侧边栏时仍保持可识别的紧凑 Logo。

2. 3D 插画
   - 新增本地 `IconfontProjectVisual` SVG 组件。
   - 用于项目列表空状态，强化“项目/测试/Agent/资产编排”的语义。

3. 动效
   - 新增 CSS 动效类，模拟 Lottie 库常见的轻量漂浮、扫描和脉冲反馈。
   - 支持 `prefers-reduced-motion` 降级，避免影响可访问性。

4. 字体层级
   - 在设计 Token 中新增 `--font-display`，以阿里妈妈字体库方向为参考，优先使用本机可用中文现代黑体，回退到系统字体。
   - 品牌、标题和摘要数字使用 display 字体栈，正文保持现有高可读字体。

## 测试策略

- 先补 `AppShell` 组件测试：
  - 品牌 Logo 存在。
  - Logo 标记为 `iconfont-cn-3d-inspired`。
  - 品牌文案仍链接到当前项目测试 Agent。
- 先补 `ProjectListScreen` 组件测试：
  - 无项目空状态出现 3D 插画。
  - 插画包含 3D、动效和来源标记。
  - 页面标题和摘要数字使用 display 字体标记。
- 实现后运行：
  - 定向 Vitest。
  - Prettier。
  - ESLint。
  - 前端 lint、typecheck、build。
  - `git diff --check`。

## 验收标准

- 平台 Logo 不再只是纯文字，视觉上具备 3D 插画质感，但不破坏专业研发工具气质。
- 项目列表空状态明显有 3D 插画和轻动效。
- 字体层级更鲜明，但正文可读性不下降。
- 不引入新运行时依赖，不使用外链素材，不提交未知授权资源。
