# TASK-20260708-001 登录直达测试 Agent 与项目列表 CRUD

## 背景

- 当前登录成功默认进入 `/projects`，但 `/projects` 会自动跳到第一个项目；当用户带有 `must_change_password` 时会被强制送到 `/account?section=security`。
- 用户要求登录后直接进入测试 Agent 页面，不要停在用户设置或账号安全页。
- 当前 `/projects` 没有独立列表页，无法集中进行项目创建、查看、重命名和归档。

## 实施范围

- 前端登录成功跳转策略。
- `/projects` 项目列表 CRUD 页面。
- 项目 API 前端封装补齐重命名和归档。
- 相关 Vitest 测试与开发记录。

## 实施步骤

1. 补充登录表单测试：默认登录、强制改密用户和账号页 returnTo 均进入首个项目测试 Agent；保留项目内深链 returnTo。
2. 补充项目列表页/组件测试：`/projects` 不再自动跳转，展示项目列表，支持创建、进入测试 Agent、重命名、归档。
3. 实现登录目的地解析：登录后读取项目列表，存在项目则进入 `/projects/{id}/test-agent`，无项目则进入 `/projects`，项目内深链继续保留。
4. 实现项目列表 CRUD UI：紧凑工作台样式，提供新建表单、列表状态、进入、编辑名称和归档操作。
5. 补齐项目 API 封装：新增 `renameProject`、`archiveProject`，使用已生成客户端与 CSRF 头。
6. 运行相关测试、格式化、lint/typecheck/build；修复发现的问题。
7. 更新开发进度记录并关闭当前任务。

## 验收标准

- 登录后不再默认进入账号安全/用户设置页；已有项目时直接进入测试 Agent。
- `/projects` 是项目管理列表页，不再加载后自动跳走。
- 项目列表可创建项目、查看项目、进入测试 Agent、重命名、归档。
- 相关测试覆盖关键行为并通过。
