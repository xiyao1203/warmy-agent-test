# 浏览器实例链路与易用性优化

## 现状确认

浏览器实例功能已有真实链路：

- 控制台前端调用 `/api/v1/projects/{project_id}/browser-profiles` 做实例 CRUD。
- 后端 `browser_profiles` 模块将实例写入 `~/.agenttest/browser-profiles/{project_id}.json`，与 Codex Browser 插件注册表共享。
- 测试计划版本配置已有 `browser_profile_id` 字段，前端会加载浏览器实例列表供选择。
- API Runner Workflow 会从用例输入或测试计划执行配置中读取 `browser_profile_id`，并传给浏览器 Activity。
- Codex Browser 插件和 Chrome Pool 能按 `browser_profile_id` 复用独立用户目录、CDP 端口和登录态。

## 问题

- 浏览器实例列表只展示表格，缺少“创建实例 → 登录保存状态 → 测试计划选择 → 运行时复用”的路径说明。
- “新建实例”等文案不够贴近日常操作。
- 没有实例时，没有直接引导用户去创建并在测试计划中使用。
- 测试计划选择“浏览器实例”时缺少来源说明和管理入口。
- 列表操作藏在更多菜单里，普通用户不容易理解下一步。

## 目标

- 显性展示浏览器实例与测试计划、测试执行的链路。
- 将操作文案调整为“新建浏览器实例 / 设置实例 / 删除”等易懂动作。
- 增加到测试计划和运行中心的快捷链路。
- 在测试计划版本向导中解释浏览器实例如何被浏览器用例复用。
- 保持已有后端 API、插件注册表和 Worker 协议不变。

## 验收

- 页面能看懂浏览器实例如何进入测试计划和运行。
- 无实例空状态给出清晰创建入口。
- 测试计划版本向导提供“去管理浏览器实例”链路。
- 相关前端测试、格式、ESLint、TypeScript 和构建通过。
