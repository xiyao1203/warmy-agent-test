# 环境凭证链路与易用性优化

## 背景

环境与凭证页面已经具备环境模板、环境版本、加密凭证绑定和测试计划环境选择能力，但页面对普通用户不够直观：操作按钮偏技术化，环境配置需要手写 JSON，凭证和测试计划之间的链路不够显性。

## 目标

- 在环境与凭证页面说明闭环：保存凭证、绑定环境、测试计划选择环境、执行时注入。
- 将“创建环境模板”“建版”等文案改成更易懂的动作。
- 创建环境和环境版本时，环境变量、公开请求头、初始状态和沙箱配置改为键值行表单。
- 在测试计划版本向导的环境选择处增加明确提示，说明环境来自“环境与凭证”页面的已发布环境。
- 保持现有 API、数据结构和加密策略不变。

## 范围

- 前端环境页和环境版本弹窗。
- 测试计划版本向导环境选择说明。
- 对应组件测试。

## 非目标

- 不新增数据库字段。
- 不改凭证加密存储方式。
- 不让前端或普通 API 读取凭证明文。
- 不改变 Worker 执行协议。

## 验收

- 页面能看懂环境凭证与测试计划、测试执行的链路。
- 用户可以不写 JSON 完成环境变量、Headers、初始状态和沙箱配置。
- 相关前端测试、格式、ESLint、TypeScript 和构建通过。
