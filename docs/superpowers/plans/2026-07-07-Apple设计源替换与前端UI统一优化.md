# Apple 设计源替换与前端 UI 统一优化

## 任务

`TASK-20260707-002：Apple 设计源替换与前端 UI 统一优化`

## 背景

用户提供 `/Users/jason/Desktop/DESIGN-apple.md`，要求替换仓库内 `docs/design.md`，并基于新的 Apple 风格设计源重新优化整体登录落地页和前端 UI。

本任务以用户当前明确指令为最高优先级。PRD 19.2 中的 Linear / Vercel 风格要求在本任务内让位于用户指定的新设计源，但仍保留工程质量、可访问性、响应式和统一组件约束。

## 范围

- 替换 `docs/design.md` 为用户提供的 Apple 风格设计源。
- 同步 `apps/web/src/styles/tokens.css` 到新设计源的 Action Blue、白/浅灰画布、近黑文字、Apple 圆角和暗色模式。
- 优化登录落地页：
  - 顶部黑色全局导航。
  - 次级 frosted 风格产品导航，右上角保留主题切换和登录入口。
  - 首屏采用全屏产品 tile 风格，避免 AI 海报感和装饰性渐变。
  - 登录通过弹窗完成。
- 收敛登录表单尺寸、圆角、按钮样式到新设计源。
- 更新登录页测试和任务台账。

## 非范围

- 不改认证 API、登录业务逻辑和权限链路。
- 不大规模重构业务页面。
- 不引入新依赖或图片资产。
- 不修改数据库、OpenAPI 或 Worker。

## 实施步骤

1. 用用户提供的 `DESIGN-apple.md` 替换 `docs/design.md`。
2. 根据新文档同步前端 CSS 设计变量。
3. 重写登录落地页为 Apple 式 tile + frosted subnav + 弹窗登录。
4. 调整登录表单到新输入框/按钮规格。
5. 更新组件测试断言。
6. 运行格式化、测试、lint、typecheck、build 和浏览器检查。
7. 更新任务记录为已完成或记录阻塞。

## 验收

- `docs/design.md` 与用户提供的文档内容一致。
- `/login` 右上角有登录按钮和主题切换。
- 点击登录打开弹窗并显示原登录表单。
- 桌面和移动视口无横向溢出、无明显遮挡。
- 前端相关测试、lint、typecheck 和 build 通过。
