# 浏览器实例启动登录删除与使用链路优化

## 现状确认

- 浏览器实例已有 CRUD，删除能力存在但藏在更多菜单中。
- 控制台只能新建名称和目标域名，没有显式“启动浏览器、人工登录、完成登录、停止浏览器”操作。
- 测试计划版本已保存 `browser_profile_id`，Worker 会把该字段传给 Codex 浏览器 Activity。
- Codex 浏览器插件会按 `browser_profile_id` 读取共享注册表并启动独立 `user-data-dir` 浏览器；但当前 Workflow 默认 `browser_mode=ephemeral`，用户不容易理解实例会如何复用。

## 问题

- 用户看不到删除按钮，误以为无法删除。
- 新建流程没有告诉用户下一步要启动浏览器并登录。
- 登录态复用依赖独立浏览器目录，但页面显示为“未登录”，理解成本高。
- 浏览器实例在哪些功能中使用不够显性。

## 计划

1. 后端浏览器实例 API 增加启动、停止、完成登录接口，启动真实本机 Chrome/Chromium 并写回运行状态；启动失败返回真实错误。
2. 前端列表外显“启动并登录 / 我已完成登录 / 停止 / 删除 / 配置到计划”操作。
3. 新建弹窗增加“创建后立即启动登录”选项和登录地址输入。
4. 页面增加“在哪使用”说明：测试计划执行配置选择实例，Codex 浏览器用例运行时复用独立浏览器目录。
5. Worker 在存在 `browser_profile_id` 时默认使用持久浏览器模式，避免选择实例后仍按临时模式执行。
6. 补充前端和 Worker 测试，运行相关验证。

## 验收

- 用户能在列表直接看到删除入口。
- 用户新建后可以直接启动浏览器登录。
- 用户完成登录后页面能显示“已确认登录”，并能停止浏览器。
- 页面能说明浏览器实例用于测试计划和 Codex 浏览器执行。
- 选择浏览器实例的测试计划运行会进入持久浏览器模式。

## 完成记录

- 浏览器实例后端已新增启动、完成登录、停止接口，并同步 OpenAPI 与生成客户端。
- 启动接口真实打开本机 Chrome/Chromium，使用实例的独立 `user-data-dir` 和 CDP 端口；失败时返回真实错误。
- 前端已将删除按钮外显，新增“启动并登录”“我已完成登录”“停止”“配置到计划”入口。
- 新建弹窗已增加登录地址和“创建后立即启动浏览器登录”选项。
- 页面已增加“这些实例用在哪儿”说明，明确测试计划和 Codex 浏览器执行会复用实例。
- Worker 已修正为存在 `browser_profile_id` 时默认使用 `persistent` 浏览器模式。
