# 2026-07-04 测试用例导入模板与 Agent 产物格式统一

## 目标

让手工新增、文件导入和测试 Agent 生成/同步的测试用例使用同一套平台标准字段，用户导入前可以直接下载完整模板并理解必填/可选字段。

## 范围

- 前端数据集导入向导：模板内容、格式说明、测试覆盖。
- 前端旧导入对话框：复用同一份模板说明，避免两个入口格式不一致。
- 后端测试 Agent 平台适配器：`datasets.create_with_cases` 与 `datasets.auto_generate_cases` 写入用例时补齐标准字段。
- 后端测试 Agent 能力入参：对 `cases` 内字段做与数据集用例一致的结构声明。

## 不做

- 不新增数据库迁移。
- 不改变 Dataset Import API 契约。
- 不实现复制、归档、从失败运行生成用例 UI。

## 实施步骤

1. 新增前端标准测试用例模板工具，输出 JSON、JSONL、CSV 三种模板和字段说明。
2. 导入向导和旧导入对话框复用模板工具，下载模板字段与新增用例表单保持一致。
3. 测试 Agent 创建/自动生成用例时，把 `initial_state`、`expected_outcome`、`security_policies`、`scenario`、`priority`、`risk_level`、`difficulty`、`test_group` 一并写入。
4. 更新测试，覆盖导入模板字段、导入对话框提示和 Agent 产物字段同步。
5. 运行数据集前端测试、后端相关单元测试、格式、Lint 和类型检查。

## 验收

- 下载模板包含 `CreateTestCaseRequest` 的核心字段。
- 导入 UI 明确必填字段和枚举取值。
- 测试 Agent 生成/同步到数据集的用例与导入/新增字段一致。
- 草稿版本仍可编辑，发布版本不可改的规则不变。
