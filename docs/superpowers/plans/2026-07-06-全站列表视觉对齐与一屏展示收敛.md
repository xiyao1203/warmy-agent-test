# 全站列表视觉对齐与一屏展示收敛

## 背景

用户反馈多个列表存在表头和值不在同一视觉中心、信息列偏左、操作列按钮拥挤换行，以及浏览器实例列表横向滚动的问题。上一轮只完成了公共表格默认居中和部分页面清理，本轮需要继续把核心业务列表实际页面收敛。

## 范围

- 公共表格组件默认表头和值居中。
- Agent、用例集、测试计划、浏览器实例、模型配置、测试账号等核心列表列宽和内容块对齐。
- 操作列按钮组在列内居中，按钮保持紧凑但不挤压换行。
- 更新相关测试，避免旧的左偏、右偏和横向滚动样式回归。

## 非目标

- 不改数据库、API 或业务流程。
- 不重做页面信息架构，只收敛列表视觉和基础可用性。

## 实施步骤

1. 统一公共 `TableHead` / `TableCell` 默认居中规则。
2. 对核心列表改为 `w-full table-fixed` 或等效固定列布局。
3. 对图标加双行文字的信息列使用“整块居中、块内左读”的布局。
4. 对操作列使用内容宽度按钮组居中，必要时将低频操作改为图标按钮。
5. 更新/补充测试并运行质量门禁。

## 验收

- 相关列表不再出现强制横向滚动或操作按钮换行。
- 表头与列值在同一列的视觉中心一致。
- 相关前端测试、类型检查和构建通过。
