# 2026-07-04 前后端遗留缺陷清零与链路验收

## 目标

清理当前仓库中可本地复现和验证的前后端遗留缺陷，尽量让质量门禁达到绿色；对于真实外部 Temporal、Model Runner、供应商模型不可用的问题，不伪造结果，而是补齐本地契约/替代测试与清晰配置要求。

## 范围

- Web：格式、Lint、类型、单元/组件测试、关键前端链路。
- Control API / Worker：Ruff、mypy、单元/契约测试、架构检查。
- 数据集/测试用例/Agent/测试计划/运行结果链路：确保本地测试覆盖新增、导入、Agent 生成同步、计划引用、结果分析的关键路径。
- 文档：开发记录不再保留可代码闭环的“遗留问题”。

## 步骤

1. 跑全量或近全量质量门禁，记录真实失败项。
2. 按失败项修复前后端代码、测试或配置。
3. 对真实外部依赖不可执行的验收，补本地契约测试或明确配置检查。
4. 复跑门禁，直到没有可本地复现的失败。
5. 更新当前任务与开发进度，完成交接。

## 验收

- 本地可执行的前后端测试、格式、Lint、类型检查通过。
- 架构/契约检查无本次引入或可修复失败。
- 文档中的遗留项只保留必须由真实外部环境验证的内容，并附本地替代验证证据。
