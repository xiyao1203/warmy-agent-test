# TapNow 画布 Agent 全链路验收计划

## 目标

使用用户提供的 TapNow 画布 Agent 作为真实目标，验证 AgentTest 平台从目标站点登录态、Agent 配置、测试资产生成、测试计划执行到结果沉淀的全链路是否可用。发现产品或代码缺陷时，按先复现、再补测试、再修复、再验证的顺序处理。

## 安全边界

- 用户提供的目标站点 JWT 只作为本地运行时凭证使用。
- 不把 JWT 写入代码、文档、测试、提交信息、终端输出或日志。
- 验证记录只记录“目标登录态可用/不可用”和失败类型，不记录凭证内容。

## 执行步骤

1. 恢复仓库和服务状态
   - 检查当前任务、开发记录、Git 状态和最近提交。
   - 检查前端、后端、Temporal、Worker、本地数据库是否在线。
   - 若服务未启动，按本地 Runbook 启动并记录可用端口。

2. 验证 TapNow 目标站点
   - 使用真实浏览器访问 TapNow 首页和用户指定的画布页面。
   - 确认画布右侧 Agent 输入区可见且可发送测试消息。
   - 若需要凭证，仅在浏览器运行时注入，不落盘。

3. 配置平台目标 Agent
   - 确认可用项目和登录态。
   - 创建或复用 TapNow 画布 Agent、版本、浏览器实例或环境配置。
   - 使用目标聊天或连接测试验证平台能访问目标页面。

4. 跑通测试资产链路
   - 通过测试 Agent 生成或手工创建最小测试用例。
   - 形成数据集和测试计划，确保执行模式适配画布浏览器探索。
   - 发布必要版本，避免使用草稿资产启动执行。

5. 执行并定位问题
   - 启动测试运行，观察 Run 状态、Trace 和 Artifact。
   - 如果返回 503，区分是 Temporal、API Runner、Model Runner 还是目标登录态缺失。
   - 如果是代码缺陷，先补充失败测试，再修复并运行匹配验证。

6. 完成记录
   - 更新 `docs/开发进度与变更记录.md`，写明真实验证结果、失败断点或修复内容。
   - 更新 `docs/当前任务.md`：完成、待验证或阻塞。

## 验收标准

- 目标画布 Agent 可被真实浏览器打开并确认 Agent 区可交互。
- 平台能创建或复用 TapNow 目标 Agent，并至少完成一次真实链路尝试。
- 若全链路成功：Run 产生可检查结果或 Trace。
- 若全链路未成功：记录首个不可跨越阻塞点、原因、风险和恢复步骤。
- 所有代码修复有对应测试或明确说明无法自动化覆盖的原因。
