# 测试执行链路与运行中心易用性优化

## 现状确认

测试执行链路已存在真实后端闭环：

- 前端 `RunCenter` 调用 `POST /projects/{project_id}/runs`，并发送 `Idempotency-Key`。
- 后端 `CreateRunHandler` 校验项目权限、幂等键、Temporal 可用性、测试计划版本和用例来源。
- 后端创建 `Run` 和 `RunCase` 后，通过 `RunOrchestrator.start` 启动 `RunWorkflow`。
- 若未配置 Temporal，`LocalRunOrchestrator` 会返回 `RunRuntimeUnavailableError`，API 以 503 明确失败，不生成假运行。
- 运行详情页已通过 SSE 订阅 `/runs/{run_id}/events`，断线后回退轮询。

## 问题

- 运行中心只要求选择“已发布版本”，没有告诉用户运行前要准备哪些资产。
- 没有已发布计划版本时，空态缺少去配置测试计划的直接入口。
- 启动运行后只刷新列表，用户还要自己找刚创建的运行。
- 运行时不可用错误直接显示英文内部错误，普通用户难理解。
- 列表操作仍是“查看”，不如“查看结果”明确。

## 目标

- 显性展示“准备计划 → 启动运行 → 查看实时进度 → 分析结果”链路。
- 没有可运行版本时，引导用户去测试计划配置并发布版本。
- 启动运行成功后自动进入运行详情页。
- 将运行时不可用等常见错误转成可理解中文。
- 保持后端 API、Temporal 编排和数据结构不变。

## 验收

- 前端测试覆盖准备链路、无版本引导、启动跳详情、中文错误提示和“查看结果”入口。
- 相关前端测试、格式、ESLint、TypeScript 和构建通过。
