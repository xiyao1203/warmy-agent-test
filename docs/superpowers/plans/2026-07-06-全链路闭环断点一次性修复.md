# 全链路闭环断点一次性修复

## 目标

修复审查中发现的真实闭环断点，使核心 Agent 测试链路和质量门禁恢复可验证状态。

## 范围

- 报告下载前后端路径不一致。
- Scorer API 直接依赖 persistence model 的架构边界问题。
- Codex 浏览器插件 storageState 采集失败。
- Python 类型检查需要显式纳入插件源码路径。
- 用户管理 cursor 分页前端只提示不操作。
- 账户两步验证属于非 MVP 能力，移除可见未开放入口，避免把未实现功能展示成平台功能。

## 验收

- `pnpm --filter @warmy/web test`
- `pnpm --filter @warmy/web typecheck`
- `pnpm --filter @warmy/web build`
- `uv run ruff check ...`
- `uv run mypy apps/control-api/src workers/api-runner/src workers/model-runner/src`
- `uv run pytest apps/control-api/tests`
- `uv run pytest workers/api-runner/tests workers/model-runner/tests`
- `PYTHONPATH=plugins/codex-browser/src uv run pytest plugins/codex-browser/tests`

## 风险

- Codex 浏览器插件依赖本地 Chrome/Playwright 环境；修复后仍需以测试结果为准。
- 两步验证不补完整后端能力，只移除未开放展示，避免扩大认证范围。
