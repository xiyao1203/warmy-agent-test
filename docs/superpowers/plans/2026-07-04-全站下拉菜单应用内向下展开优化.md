# 全站下拉菜单应用内向下展开优化

## 现状

- 多个页面直接使用浏览器原生 `<select>`。
- 原生菜单由浏览器/操作系统决定弹出方向，可能向上覆盖内容，且视觉风格不统一。
- 运行中心、测试计划版本、Agent、数据集、用户、模型配置等模块都有下拉。

## 目标

- 新增共享 `DropdownSelect` 组件。
- 菜单由应用内 DOM 渲染，点击后固定从控件下方展开。
- 保持原有 `value/onChange/disabled/aria-label` 交互，减少业务代码改动。
- 替换 Web 生产源码中的原生 `<select>`。

## 计划

1. 创建共享 `DropdownSelect`。
2. 替换运行中心、测试计划、Agent、数据集、评分器、实验、门禁、安全、用户、模型配置等页面下拉。
3. 保留隐藏原生 select 作为表单状态和测试兼容层，但用户可见操作使用应用内菜单。
4. 更新或补充测试，覆盖菜单向下展开和现有表单变更。
5. 运行搜索、格式、ESLint、TypeScript、Vitest 和构建。

## 验收

- 生产页面不再直接显示原生下拉菜单。
- 点击下拉后菜单出现在控件下方。
- 现有表单测试仍可变更选项。
- 前端测试和构建通过。

## 完成记录

- 已新增 `DropdownSelect`，应用内菜单使用 `side="bottom"` 和 `avoidCollisions={false}` 固定向下展开。
- 已替换 Web 业务页面中的可见原生 `<select>`，运行中心、测试计划、Agent、用例、导入导出、评分器、实验、门禁、安全、测试 Agent、模型、测试账号和用户管理均改用共享组件。
- `rg -n "<select|</select>" apps/web/src -g "*.tsx"` 仅命中 `DropdownSelect` 内部隐藏兼容层。
- 组件测试覆盖应用内菜单打开、`data-side="bottom"` 和隐藏 select 表单兼容。
- 已通过重点测试、全量前端测试、TypeScript、ESLint、Prettier 和构建。
- 内置浏览器未登录时会跳转登录页，因此未直接进入项目页截图；本次以组件级方向断言和全量回归作为验收证据。
