# 用例集与测试计划列表链路体验优化

## 背景

用户反馈用例集列表仍然显示“查看”，测试计划列表也缺少面向普通用户的清晰链路说明。当前详情页已经支持测试计划版本选择 Agent、用例集版本、环境、评分器和门禁，但列表页没有把这些后续动作显性化。

## 目标

- 用例集列表入口从“查看”改为明确的“管理用例”。
- 测试计划列表入口从“查看”改为明确的“配置计划”。
- 测试计划列表文案说明计划与 Agent、用例集、环境、执行、评分和门禁的关系。
- 空状态提供去创建用例集的链路，降低从测试计划到测试资产准备的理解成本。
- 保持现有 API 和数据库结构不变。

## 范围

- 前端：
  - `apps/web/src/features/datasets/dataset-list.tsx`
  - `apps/web/src/features/test-plans/test-plan-list.tsx`
  - 相关组件测试
- 文档：
  - `docs/当前任务.md`
  - `docs/开发进度与变更记录.md`

## 非目标

- 不改后端 API。
- 不新增数据库字段。
- 不改变测试计划详情页版本向导的数据模型。

## 验收

- 用例集列表不再出现“查看用例集”式模糊入口。
- 测试计划列表展示清晰的配置入口和资产链路提示。
- 测试计划空状态能引导用户先准备用例集或创建计划。
- 相关前端测试、格式检查、ESLint、TypeScript 和构建通过。
